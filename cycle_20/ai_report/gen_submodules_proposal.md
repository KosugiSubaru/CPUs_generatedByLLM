教育用シングルサイクルCPU（16ビット）として、ISA定義に基づいた最適なモジュール構成と動作フローを提案します。

---

### 1. CPU構成サブモジュール一覧

CPU内部を以下の7つの主要モジュールに分割することを提案します。

| モジュール名 | 役割 |
| :--- | :--- |
| **1. Program Counter (PC)** | 次に実行する命令のアドレスを保持するレジスタ。 |
| **2. Control Unit** | 命令の下位4ビット（opcode）をデコードし、各モジュールへの制御信号（ALUの演算種別、レジスタ書き込み許可、メモリ選択など）を生成する。 |
| **3. Register File** | 16個の16ビットレジスタ。`rs1`, `rs2`の読み出しと、`rd`への書き込みを行う。レジスタ0番（R0）は常に0を出力する。 |
| **4. Immediate Extender** | 命令形式（I-type, Jump系など）に合わせて即値フィールドを切り出し、16ビットに符号拡張する。 |
| **5. ALU (Arithmetic Logic Unit)** | 算術演算、論理演算、シフト演算を行う。演算結果とともに、フラグ（Z, N, V）を出力する。 |
| **6. Flag Register** | ALUが出力したフラグ（Z, N, V）を1クロック保持する。条件分岐命令は「現在のALU結果」ではなく「このレジスタに保持された前回の結果」を参照する。 |
| **7. PC Control / Next PC Logic** | 通常のインクリメント（PC+2）、分岐（PC+imm）、ジャンプ（PC+imm / rs1+imm）のいずれかを、制御信号とフラグの状態に基づいて選択する。 |

---

### 2. CPUの動作プロセス（相互作用）

シングルサイクルCPUとして、1クロックの間に以下のフローが並列・連鎖的に発生します。

#### ① Fetch (命令取得)
- **PC**が保持しているアドレスが命令メモリ（imem）に送られ、16ビットの命令（`i_instr`）が取り出されます。

#### ② Decode (解読) & Read (読み出し)
- **Control Unit**が命令の`opcode`を見て、「これは加算か？」「メモリ読み込みか？」を判断し、システム全体のセレクタを切り替えます。
- **Register File**から`rs1`, `rs2`番台のデータが読み出されます。
- **Immediate Extender**が、命令内の特定のビット列を抜き出し、16ビットの即値に変換します。

#### ③ Execute (実行)
- **ALU**が、レジスタから読みた値や即値を用いて演算（加算、論理演算、シフトなど）を行います。
- 同時に、演算結果からフラグ（Z:ゼロ、N:負、V:オーバーフロー）を計算します。
- **Flag Register**は、*次のクロックの立ち上がり*でこれらのフラグを保存します。これにより、条件分岐命令は「1つ前の命令の実行結果」に基づいて分岐判定を行います（ISA仕様通り）。

#### ④ Memory (データアクセス)
- `load`命令なら、ALUで計算されたアドレスを使ってデータメモリ（dmem）から値を読み出します。
- `store`命令なら、`rs2`の値をデータメモリに書き込みます（`o_dmem_wen`が有効になる）。

#### ⑤ Write Back (書き戻し)
- 最終的な結果（ALUの演算結果、またはメモリから読み出した値、またはPC+2）を、**Register File**の`rd`番レジスタに書き込みます。

#### ⑥ Next PC (次アドレス決定)
- **PC Control**モジュールが、分岐条件（`bz`, `blt`）が満たされているか、またはジャンプ命令（`jal`, `jalr`）かを判断します。
- 条件が満たされていれば`PC + imm`などを、そうでなければ`PC + 2`を選択し、次のサイクルに向けてPCを更新します。

---

### 教育的視点でのポイント

この階層化により、学生は以下を視覚的に理解できます。
1.  **データパスの流れ:** 命令が左（PC）から右（書き戻し）へ流れていく様子。
2.  **フラグの遅延:** 条件分岐が「1クロック前の状態」を見るための、Flag Registerという物理的な壁の存在。
3.  **opcodeの役割:** 下位4ビットがいかにしてCPU全体の「スイッチ（制御信号）」を動かしているか。
4.  **即値の扱い:** 命令の種類によって、即値として取り出すビット位置が異なること（符号拡張の重要性）。
