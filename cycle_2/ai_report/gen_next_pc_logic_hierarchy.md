次アドレス生成ロジック（`next_pc_logic`）は、現在のPC、即値、レジスタ値、およびフラグ状態を組み合わせて、次に実行すべき命令のアドレスを決定する「判断と計算」のブロックです。

単なる加算器やセレクタの組み合わせではなく、**「なぜこのアドレスが選ばれたのか」**が回路図から直感的に理解できるように階層化・パターン構造化します。

---

### 1. 提案するサブモジュール構成

| レベル | ファイル名（モジュール名） | 役割 |
| :--- | :--- | :--- |
| **L1** | `next_pc_logic.v` | **最上位結合**: 加算器、条件判定器、MUXを接続し、次PCのアドレス値を確定させる。 |
| **L2** | `next_pc_logic_adder_16bit.v` | **アドレス計算器**: `generate`文を用い、16個の全加算器で構成された加算ユニット。 |
| **L2** | `next_pc_logic_cond_eval.v` | **分岐判定**: 命令（blt, bz等）とフラグ（Z, N, V）を照合し、分岐の成否を1bitで出力。 |
| **L2** | `next_pc_logic_mux4_16bit.v` | **次PC選択**: `generate`文を用い、4つのアドレス候補から1つを選択する16bit MUX。 |
| **L3** | `next_pc_logic_full_adder.v` | **最小単位(演算)**: 1ビットの全加算器。 |
| **L3** | `next_pc_logic_mux2_1bit.v` | **最小単位(選択)**: 1ビットの2入力マルチプレクサ。 |

---

### 2. 階層化・パターン構造化のポイント

#### ① アドレス計算の分離 (`next_pc_logic_adder_16bit.v`)
相対分岐（PC + imm）と絶対分岐（rs1 + imm）、および通常のインクリメント（PC + 2）の計算を明示的に分けます。
*   **パターン構造**: `generate`文で `full_adder` を16個直列に繋ぎ、キャリーの伝搬を視覚化します。
*   **視覚的効果**: 回路図上で、現在のPCやレジスタ値が即値と合流し、新しいアドレスへと変換される様子が「計算機の塊」として独立して見えます。

#### ② 条件評価ロジックの独立 (`next_pc_logic_cond_eval.v`)
ISA定義の「条件分岐命令はフラグを参照して分岐する」というロジックを独立させます。
*   **役割**: `blt` (N ^ V) や `bz` (Z) の条件を判定します。
*   **視覚的効果**: 複雑な条件判定（XORやANDゲートの組み合わせ）が「判定器」という一つのブロックに集約され、デコード信号とフラグがどのように組み合わさって分岐を決めるかが一目で分かります。

#### ③ 選択網の構造化 (`next_pc_logic_mux4_16bit.v`)
最終的な次PCを選ぶ4入力セレクタを、ビットごとにパターン化します。
*   **パターン構造**: `generate`文を用いて、16ビット分並列に `mux2_1bit` を配置した構造にします。

---

### 3. モジュール名リスト

submodules:
next_pc_logic: PCのインクリメント、相対分岐、絶対ジャンプのいずれを行うか判断し、次サイクルのPC値を算出・選択する最上位モジュール。
next_pc_logic_adder_16bit: generate文を用いて1ビット全加算器を16個接続し、アドレス加算（PC+2、PC+imm、rs1+imm）を行う16ビット加算器。
next_pc_logic_cond_eval: 現在のフラグ（Z, N, V）と命令の種類（blt, bz）を照合し、分岐を「取る（Taken）」か「取らない（Not Taken）」かを判定するロジック回路。
next_pc_logic_mux4_16bit: generate文を用いて1ビット2入力MUXを組み合わせ、4つのアドレス候補（順次、相対、絶対、初期）から最終的な次PCを選択する16ビット・4入力セレクタ。
next_pc_logic_full_adder: 加算器の最小単位となる、キャリー入出力を備えた1ビット全加算器。
next_pc_logic_mux2_1bit: 選択信号に応じて2つの入力から1つを選択する、セレクタの最小構成単位。

---

### 4. 動作プロセスの簡潔な説明

1.  **Parallel Addition**: 現在の `PC` に対して、「+2」「+imm」、およびレジスタからの「`rs1` + `imm`」の3つの計算が、独立した `adder_16bit` ユニットによって並行して行われます。
2.  **Condition Evaluation**: 同時に `cond_eval` がフラグの状態を確認し、現在の命令が分岐条件を満たしているかを判定します。
3.  **Final Selection**: `control_unit` からの制御信号と `cond_eval` の判定結果を組み合わせて `mux4_16bit` が切り替わり、次のクロックで `program_counter` が保持すべきアドレスを1つ出力します。
