教育用シングルサイクルCPUの設計において、ISA定義に基づいた1階層下のサブモジュール構成を提案します。

この構成は、命令の「フェッチ → デコード → 実行 → メモリ → 書き戻し」というデータの流れ（データパス）を視覚的に分離し、各モジュールの役割を明確にすることを重視しています。

---

### 1. 提案するサブモジュール一覧

CPU内部を以下の8つの主要モジュールに分割します。

| モジュール名 | 役割・機能説明 |
| :--- | :--- |
| **PC (Program Counter)** | 次に実行する命令のアドレス（16bit）を保持するレジスタ。 |
| **Instruction Decoder** | 16bitの命令を、opcode, rd, rs1, rs2, immなどのフィールドに分解する。 |
| **Control Unit** | opcodeを解析し、各部へのセレクタ信号、レジスタ書き込み許可（WE）、メモリ書き込み許可などを出力する「司令塔」。 |
| **Register File** | 16bit×16本のレジスタ群。rs1, rs2の読み出しと、rdへの書き込みを行う。R0は常に0を出力する。 |
| **ALU (Arithmetic Logic Unit)** | 加減算、論理演算、シフト演算を行う。演算結果とともに、フラグ（Z, N, V）を生成する。 |
| **Flag Register** | ALUが生成したフラグを1クロック保持する。条件分岐命令（blt, bz）が参照する。 |
| **Immediate Generation** | 命令形式に合わせて、即値フィールドを抽出し、16bitに符号拡張する。 |
| **Next PC Logic** | PC+2（通常時）、PC+imm（分岐・JAL）、rs1+imm（JALR）の中から、次にPCにセットすべき値を選択する。 |

---

### 2. 各モジュールの相互作用と動作フロー

シングルサイクルCPUとして、1クロックの間に以下の流れで処理が行われます。

#### ① 命令フェッチ (Fetch)
- **PC**が保持しているアドレスを命令メモリ（imem）へ送ります。
- imemから16bitの**Instruction**がCPUに入力されます。

#### ② デコードとレジスタ読み出し (Decode)
- **Instruction Decoder**が命令を分割します（例：`opcode = inst[3:0]`）。
- **Control Unit**がopcodeに基づき、実行すべき演算や書き込みの有無を判断します。
- **Register File**は、指定された`rs1`, `rs2`のアドレスからデータを出力します。
- **Immediate Generation**は、命令に含まれる即値を符号拡張して準備します。

#### ③ 実行 (Execute)
- **ALU**は、レジスタから読み出した値（または即値）を用いて演算を行います。
- **Flag Register**は、演算結果から得られたフラグ（Z, N, V）を更新します（※このフラグは「次の命令」の分岐判断に使われます）。
- 分岐命令の場合、**Flag Register**に保持されている「1クロック前」のフラグと、**Control Unit**の指令を照らし合わせ、分岐するかどうかを決定します。

#### ④ メモリ・書き戻し (Memory / Write Back)
- **Load/Store命令の場合**: ALUの演算結果（rs1 + imm）をアドレスとしてデータメモリ（dmem）にアクセスします。
- **書き戻し**: 最終的に「ALUの結果」「メモリからの読み出しデータ」「PC+2（JAL用）」のいずれかをセレクタで選び、**Register File**の`rd`へ書き込みます。

#### ⑤ PC更新 (PC Update)
- **Next PC Logic**が、分岐条件やジャンプ命令の有無に基づき、次のクロックでのPCの値を決定します。
    - 通常：`PC + 2`
    - 分岐成立・JAL：`PC + imm`
    - JALR：`rs1 + imm`

---

### 3. この構成の教育的ポイント

1.  **フラグの遅延**: 「条件分岐は1クロック前のフラグを参照する」という仕様を、独立した**Flag Register**モジュールを置くことで、「今計算しているフラグ」と「今分岐判断に使っているフラグ」の違いを視覚的に示せます。
2.  **ゼロレジスタの明示**: **Register File**内で`addr==0`のときに強制的に0を出力する回路を含めることで、ISAの制約がどうハードウェアに反映されるかを学べます。
3.  **即値の符号拡張**: 命令形式（Rタイプ、Iタイプ、Jタイプ等）によって即値の位置や長さが異なるため、**Immediate Generation**を独立させることで、ビットの切り出しと拡張の仕組みを整理できます。

この構成で設計を進めることで、各モジュールがVerilogのcase文やassign文と1対1で対応しやすくなり、初心者にとっても理解しやすいCPU回路となります。
