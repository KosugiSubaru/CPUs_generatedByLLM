「教育用CPU」として、レジスタの集合（レジスタファイル）がどのようにデータを保持し、アドレスによってどのようにデータが選択・更新されるかを視覚化するため、以下の階層化を提案します。

特に、16個のレジスタから特定の1つを選ぶ「16対1セレクタ（MUX）」と、書き込み先を特定する「4対16デコーダ」を段階的に構成することで、大規模な論理回路が小さな部品の組み合わせであることを示します。

---

### 1. 階層構造の概要

`register_file` モジュールを以下の3つのレベルに階層化します。

*   **Level 2 (Primitive):** 最小単位。
    *   1ビット保持回路（イネーブル付DFF）、1ビット4入力セレクタ、2対4デコーダ。
*   **Level 1 (Structural):** 機能をまとめた16ビット/16レジスタ対応ブロック。
    *   **16ビットレジスタ:** DFFを16個並列配置。
    *   **16対1セレクタ:** 4対1セレクタを木構造で組み合わせ、16個の入力から1ビットを選ぶ。
    *   **16対1セレクタ(16ビット幅):** 1ビット16対1セレクタを16個並列配置。
    *   **4対16デコーダ:** 2対4デコーダを組み合わせて構成。
*   **Level 0 (Top):**
    *   15個の16ビットレジスタ（R1～R15）をインスタンス化。
    *   R0は「常に0を出力する」というISAの制約を配線で実現。

---

### 2. サブモジュールのリストと役割

| 階層 | ファイル名 | 役割の説明 |
| :--- | :--- | :--- |
| **Level 2** | **register_file_dff_en.v** | 書き込み許可信号(EN)付きの1ビットD型フリップフロップ。 |
| **Level 2** | **register_file_mux_4to1_1bit.v** | 4つの1ビット入力から1つを選ぶ基本セレクタ。 |
| **Level 2** | **register_file_decoder_2to4.v** | 2ビット入力を4本の有効信号に変換する基本デコーダ。 |
| **Level 1** | **register_file_reg_16bit.v** | `generate`文を用いて `dff_en` を16個並列配置した16ビット保持レジスタ。 |
| **Level 1** | **register_file_mux_16to1_1bit.v** | `mux_4to1_1bit` を組み合わせて構成した1ビット幅の16入力セレクタ。 |
| **Level 1** | **register_file_mux_16to1_16bit.v** | `generate`文を用いて `mux_16to1_1bit` を16個配置した16ビット幅の16入力セレクタ。 |
| **Level 1** | **register_file_decoder_4to16.v** | `decoder_2to4` を組み合わせて構成した、書き込み先レジスタ選択用の4対16デコーダ。 |
| **Level 0** | **register_file.v** | 15個のレジスタ、2組の読み出し用16ビットMUX、1組のデコーダを統合する最上位。 |

### 3. 階層化とパターン構造化の意図

1.  **R0 = 0 の可視化**:
    `register_file.v` において、レジスタR0のアドレスには実体としてのレジスタを接続せず、MUXの入力に直接「0」を接続します。これにより、回路図上で「R0は定数として扱われている」ことが一目でわかります。
2.  **マルチポート読み出しの再現**:
    rs1とrs2の2系統の読み出しポートを実現するために、`mux_16to1_16bit` を2つ並列にインスタンス化します。これにより、同じレジスタ群から同時に2つのデータを引き出す物理的な構造が理解できます。
3.  **ビットスライス構造**:
    16ビット幅のMUXやレジスタを `generate` 文で1ビット単位から構築することで、バス幅という概念が1ビットの並列処理であることを示します。

---

### 4. 相互作用の説明

1.  **書き込み (Write)**:
    `rd_addr` を `decoder_4to16` で解析し、さらに `rd_en`（書き込み許可）との論理積をとることで、16本のレジスタ更新信号のうち1本だけを有効にします。
2.  **保持 (Storage)**:
    有効になった更新信号を受け取った `reg_16bit` だけが、クロックの立ち上がりで `rd_data` を内部に取り込みます。
3.  **読み出し (Read)**:
    全レジスタ（R1～R15）の出力と、定数0（R0用）が、2つの大きな `mux_16to1_16bit` に同時に入力されます。`rs1_addr` と `rs2_addr` に従って、それぞれ独立したデータが出力されます。
