教育用シングルサイクルCPUとして、命令セット（ISA）の各要素と回路構成が1対1で対応するように設計することが重要です。

提案するCPUの内部サブモジュール構成と、その動作の流れを以下に示します。

---

### 1. サブモジュール構成案

CPUの内部を以下の7つの主要モジュールに分割することを提案します。

| モジュール名 | 役割 |
| :--- | :--- |
| **1. Program Counter (PC)** | 次に実行する命令のメモリアドレス（`o_addr_to_imem`）を保持するレジスタ。 |
| **2. Instruction Decoder** | 命令コード（`i_instr`）を分解し、各モジュールへの制御信号（Opcode、レジスタ番号、即値など）を生成する。 |
| **3. Register File** | 16個の16bitレジスタを管理。2つの読み出しポート（rs1, rs2）と1つの書き込みポート（rd）を持つ。R0は常に0を返す。 |
| **4. Immediate Generator** | 命令内の異なるビット位置にある即値（4bit/8bit/12bit）を取り出し、符号拡張を行って16bitデータに変換する。 |
| **5. ALU (Arithmetic Logic Unit)** | 算術論理演算を実行。演算結果とともに、フラグ（Z, N, V）を出力する。 |
| **6. Flag Register** | ALUが出力したフラグ（Z, N, V）をクロック立ち上がりで保存する。条件分岐命令は、この保存された「1サイクル前のフラグ」を参照する。 |
| **7. PC Control Unit** | 次のPC値を決定する。通常インクリメント（PC+2）、相対分岐（PC+imm）、絶対分岐（rs1+imm）の切り替えを行う。 |

---

### 2. CPUの動作フロー（相互作用）

シングルサイクルCPUとして、1クロック内で以下のステップが同時に進行します。

#### ① 命令フェッチ & デコード (Fetch & Decode)
- **PC**が保持するアドレスが命令メモリ（imem）へ送られ、`i_instr`として命令が届きます。
- **Instruction Decoder**が命令を解析し、レジスタ番号を**Register File**に、即値部分を**Immediate Generator**に送ります。同時に、ALUの演算種類やメモリ書き込みの有無などの制御信号を出力します。

#### ② オペランド読み出し (Operand Fetch)
- **Register File**から指定された `rs1`, `rs2` のデータが読み出されます。
- **Immediate Generator**は、命令の形式に合わせて即値を16bitに符号拡張します。

#### ③ 実行・計算 (Execute)
- **ALU**は、レジスタ値または即値を用いて演算（加減算、論理演算、シフト等）を行います。
- 演算の結果、新しいフラグ（Z, N, V）が生成されます。これらはクロックの立ち上がりで**Flag Register**に保存されます（次の命令が利用するため）。

#### ④ メモリアクセス (Memory Access)
- **load/store命令の場合**: ALUで計算されたアドレス（rs1 + imm）が `o_addr_to_dmem` として出力されます。
- `store` なら `rs2` の値がデータメモリへ書き込まれ、`load` ならデータメモリからの値 `i_data_from_dmem` がCPU内に取り込まれます。

#### ⑤ 書き戻し (Write Back)
- **Register File**の書き込みポート（rd）に対して、以下のいずれかのデータを選択して書き込みます。
    - ALUの演算結果（通常の演算）
    - データメモリからの読み出し値（load命令）
    - PC + 2 の値（jal / jalr命令：戻りアドレスの保存）

#### ⑥ 次のPCの決定 (PC Update)
- **PC Control Unit**が、現在の命令と**Flag Register**の値を参照します。
    - `bz` かつ Zフラグが1の場合、または `blt` かつ N xor V が1の場合：`PC <- PC + imm`
    - `jal` 命令：`PC <- PC + imm`
    - `jalr` 命令：`PC <- rs1 + imm`
    - それ以外：`PC <- PC + 2`（次の命令へ）

---

### 3. 教育的な視点でのポイント

- **フラグの遅延**: ISA定義にある「1クロック前のフラグを参照する」という仕様は、ALU出力を直接分岐判定に使わず、一度レジスタ（Flag Register）を介することで、回路図上で「状態の保持」を視覚的に説明しやすくなります。
- **マルチプレクサ (MUX) の可視化**: 上記の「書き戻しデータの選択」や「ALU入力の選択（rs2か即値か）」の部分にMUXを配置することで、命令ごとにデータの流れ（データパス）がどう切り替わるかを直感的に理解できるようになります。
- **R0 = 0 の固定**: レジスタファイルの内部でR0への入力を無視し、出力を0に固定する構成は、ハードウェアによる制約の実装例として良い教材になります。
