教育用シングルサイクルCPUにおいて、レジスタファイルの「16ビット×16個の記憶保持」と「R0=常に0」という挙動を視覚化するため、以下の5段階の階層構造とパタン構造化を提案します。

この設計では、16個のレジスタが物理的に並び、デコーダによって書き込み先が選ばれ、マルチプレクサによって読み出しデータが選ばれる様子を、回路図上でブロックとして明確に表現します。

---

### 階層構造図

```text
register_file (Level 3: レジスタファイル全体)
├── register_file_decoder_4to16 (Level 2: 書き込みデコーダ)
├── register_file_reg_16bit (Level 2: 16bit汎用レジスタ)
│   └── register_file_bit (Level 1: 1bit D-FF with Enable)
├── register_file_zero_reg_16bit (Level 2: ゼロレジスタ)
└── register_file_mux_16to1_16bit (Level 2: 16進1出力セレクタ)
    └── register_file_mux_16to1_1bit (Level 1: 1bitセレクタ)
```

---

### 各ファイルの役割と階層レベルの説明

#### Level 1: ビットレベル・コンポーネント
最小単位のロジックを定義し、16ビットへの展開準備をします。

1. **register_file_bit.v**
   - **役割**: 書き込み有効信号（Enable）付きの1ビットDフリップフロップ。
   - **仕様**: `wen`が1のときのみ値を更新します。
2. **register_file_mux_16to1_1bit.v**
   - **役割**: 16個の入力から1つを選択する1ビットセレクタ。
   - **仕様**: 読み出しポート（rs1, rs2）の選択ロジックの最小単位です。

#### Level 2: パタン構造化・機能ブロック
`generate`文を使用し、Level 1のモジュールを並列展開して「データ幅（16bit）」や「デコード論理」を構成します。

3. **register_file_reg_16bit.v**
   - **役割**: **register_file_bit**を16個並べた、1つのレジスタ。R1～R15に使用されます。
4. **register_file_zero_reg_16bit.v**
   - **役割**: ISA定義の「R0は常に0」を視覚化するための専用モジュール。
   - **仕様**: 内部にフリップフロップを持たず、常に16'h0000を出力します。回路図上で他のレジスタと区別して表示されます。
5. **register_file_mux_16to1_16bit.v**
   - **役割**: **register_file_mux_16to1_1bit**を16個並べ、16個のレジスタから1つの16ビット値を選択します。
6. **register_file_decoder_4to16.v**
   - **役割**: 4ビットの書き込みアドレス（rd）を、16本の書き込み有効信号線にデコードします。

#### Level 3: トップレベル・サブモジュール
7. **register_file.v**
   - **役割**: 全てのコンポーネントを統合します。
   - **パタン構造化**: `generate`文を用いて、R1～R15の15個の汎用レジスタを一括でインスタンス化し、R0のみ独立して接続します。
   - **入出力**: rs1/rs2の読み出し、rdの書き込みを管理します。

---

### 提案する実装のパタン構造化の例 (Verilogイメージ)

`register_file.v` トップにおいて、レジスタの集合を以下のように記述することで、回路図上での整列とR0の特殊性を際立たせます。

```verilog
// register_file.v の構成例
// R0のインスタンス化 (常に0)
register_file_zero_reg_16bit u_reg0 (
    .o_q (w_reg_data[0])
);

// R1〜R15のインスタンス化 (15個の汎用レジスタを整列)
genvar i;
generate
    for (i = 1; i < 16; i = i + 1) begin : gen_regs
        register_file_reg_16bit u_reg (
            .i_clk   (i_clk),
            .i_rst_n (i_rst_n),
            .i_wen   (w_wen_lines[i]), // デコーダからの信号
            .i_d     (i_rd_data),
            .o_q     (w_reg_data[i])
        );
    end
endgenerate
```

### この階層化による視覚的効果

1.  **「R0=0」の明示**: R0だけがレジスタ（FF）ではなく、定数出力ブロックに繋がっていることが回路図上で一目でわかります。
2.  **アドレスデコードと選択の分離**: 「rdアドレスで書き込み先を選ぶ」「rs1/rs2アドレスで読み出し元を選ぶ」という2つの独立した動作が、デコーダとマルチプレクサという明確に分かれたブロックとして表現されます。
3.  **拡張性の理解**: 16ビットという幅が、1ビットのコンポーネントを16個並べることで作られている（ビットスライス構造）ことが視覚的に理解できます。
