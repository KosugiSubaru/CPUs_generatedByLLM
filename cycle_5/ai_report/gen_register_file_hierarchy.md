教育用プロセッサにおいて、**Register File**は「データの置き場所」を視覚化する重要なモジュールです。16個のレジスタが整然と並び、指定された番号に従ってデータが吸い上げられたり、書き込まれたりする様子を強調するため、以下の階層化とパターン構造化を提案します。

特に、**「R0は常に0」**というISAの制約を回路図上で明示的に表現する構造を採ります。

---

### 1. 階層構造の概要

レジスタファイルを「書き込み選択（デコーダ）」「データ保持（ストレージ）」「読み出し選択（マルチプレクサ）」の3つの独立したブロックに分離します。

- **Level 1: Top Module**
    - `register_file`: 全体を統合するトップ。
- **Level 2: Intermediate Modules**
    - `register_file_write_decoder`: 書き込み先アドレス（4bit）を16本の有効信号に変換する。
    - `register_file_storage`: レジスタ本体の集合体。16個の16bitレジスタを並べる。
    - `register_file_mux_16to1`: 16個の入力から1つを選択する読み出し用大容量セレクタ。
- **Level 3: Logic/Storage Units (Pattern Units)**
    - `register_file_cell_16bit`: 16ビットのレジスタ1本分。
    - `register_file_cell_1bit`: 1ビットの保持回路（D-FF）。
    - `register_file_mux_4to1`: 4入力マルチプレクサ（16to1を構成する基本要素）。

---

### 2. サブモジュールのリストと役割

| ファイル名 / モジュール名 | 階層 | 役割の説明 |
| :--- | :---: | :--- |
| **register_file.v** | 1 | **最上位構成:** デコーダ、ストレージ、読み出しMUXを結合。CPUトップからの読み書き要求を処理する。 |
| **register_file_write_decoder.v** | 2 | **書き込みアドレスデコーダ:** 4bitのアドレスを16bitのOne-hot信号に変換。さらに `RegWrite` 信号とANDを取り、各レジスタの書き込み有効化信号を作る。 |
| **register_file_storage.v** | 2 | **レジスタ集合体:** `generate` 文を用いて `register_file_cell_16bit` を16個並べる。**R0については、出力を常に0に固定する特殊処理**を行い、ISAの制約を可視化する。 |
| **register_file_cell_16bit.v** | 3 | **16bitレジスタ:** `generate` 文を用いて `register_file_cell_1bit` を16個並べる。 |
| **register_file_cell_1bit.v** | 4 | **1bit保持回路:** 書き込み有効信号(Enable)付きのD-フリップフロップ。レジスタの最小単位。 |
| **register_file_mux_16to1.v** | 2 | **16入力マルチプレクサ:** `register_file_mux_4to1` を階層的に組み合わせて構成。16個のレジスタ出力から1つを選択する。 |
| **register_file_mux_4to1.v** | 3 | **4入力マルチプレクサ:** 4つのデータから1つを選択する。大規模なMUXをパターン化するための基本部品。 |

### 3. パターン構造化と視覚化のポイント

#### ① R0の特殊性の可視化
`register_file_storage` の中で `generate` 文を使用する際、インデックス `i=0` の場合だけはレジスタをインスタンス化せずに定数 `0` を接続するか、あるいはインスタンス化した上で出力を `0` に捨てる構造にします。これにより、合成後の回路図で「R0だけ配線が他と違う」ことが一目でわかります。

#### ② ビットスライスの強調
`register_file_cell_16bit` で1ビットのセルを16個並べることで、回路図上に16本の同じ回路が並びます。これは「データ幅」という概念を物理的な並びとして理解するのに役立ちます。

#### ③ 読み出しパスの分離（Dual Port Read）
このISAでは `rs1` と `rs2` の2つを同時に読み出す必要があるため、`register_file_mux_16to1` を2つインスタンス化します。回路図上では、ストレージブロックから2つの大きなMUXへ向かって大量の配線が分岐して伸びる様子が描かれ、マルチポート読み出しの構造が明確になります。

### 4. 教育的な効果
合成後の回路図では、中央に巨大な **Storage Block** があり、その左側に **Write Decoder**、右側に2つの **Read MUX** が配置されます。「アドレスで場所を決める」「中身を保持する」「欲しいデータを選び出す」というレジスタファイルの3要素がブロックとして分離され、学生が混乱せずにデータパスを追えるようになります。
