教育用プロセッサのコアとなる**ALU**は、データの加工と計算の状態（フラグ）を生成する最重要コンポーネントです。

ISAで定義された算術演算、論理演算、シフト演算、およびフラグ生成（Z, N, V）を回路図上で明確に分離し、計算の仕組みを視覚化するため、以下の階層化とパターン構造化を提案します。

---

### 1. 階層構造の概要

ALUを「演算実行ユニット群（算術・論理・シフト）」、「結果選択（MUX）」、「ステータス生成（フラグ）」の3要素に分解します。

- **Level 1: Top Module**
    - `alu`: 各演算ユニットを並列に動かし、最終的な結果とフラグを統合出力する。
- **Level 2: Functional Units**
    - `alu_arithmetic`: 加減算を担当。フラグ N, V の計算の基礎となる和とキャリーを生成。
    - `alu_logical`: AND, OR, XOR, NOT のビット単位演算。
    - `alu_shifter`: SRA, SLA のシフト演算。
    - `alu_mux_result`: 演算モードに基づき、複数の演算結果から1つを選択する。
    - `alu_flag_z_calc`: 最終結果が 0 かどうかを判定して Z フラグを出力する。
- **Level 3: Logic/Pattern Units**
    - `alu_adder_nbit`: 16ビットの加算器。`generate` 文でビットごとの加算を繋ぐ。
    - `alu_adder_1bit`: 全加算器（Full Adder）。演算の最小単位。

---

### 2. サブモジュールのリストと役割

| ファイル名 / モジュール名 | 階層 | 役割の説明 |
| :--- | :---: | :--- |
| **alu.v** | 1 | **最上位構成:** 各種演算ユニットを並列に結合。計算結果とフラグ（Z, N, V）を統合して出力する。 |
| **alu_arithmetic.v** | 2 | **算術演算器:** 加算と減算を実行。減算時は2の補数（反転+1）を使い `alu_adder_nbit` を呼び出す。フラグ N, V のロジックを含む。 |
| **alu_adder_nbit.v** | 3 | **16bit加算器:** `generate` 文を用いて `alu_adder_1bit` を16個直列に接続（Ripple Carry形式）。 |
| **alu_adder_1bit.v** | 4 | **1bit全加算器:** 2つの入力とキャリー入力から、和とキャリー出力を生成する最小の計算ブロック。 |
| **alu_logical.v** | 2 | **論理演算器:** ビットごとの AND, OR, XOR, NOT を実行。 |
| **alu_shifter.v** | 2 | **シフト演算器:** 指定されたビット数（rs2）だけ算術右シフト、算術左シフトを実行する。 |
| **alu_mux_result.v** | 2 | **結果選択器:** 演算モード（alu_mode）に従い、加減算、論理、シフトのいずれの結果を rd に書き戻すか選択する。 |
| **alu_flag_z_calc.v** | 2 | **ゼロ判定器:** 16ビットの演算結果を入力とし、全てのビットが 0 であることを検知して Z フラグを出力する。 |

### 3. パターン構造化と視覚化のポイント

#### ① 演算の「並列性」の可視化
ALU内部では、算術、論理、シフトの各ユニットが**常に同時に計算**されており、最後に `alu_mux_result` で必要なものだけをピックアップしている様子を回路図上で示します。これはプロセッサの動作を理解する上で非常に重要です。

#### ② 加算器の構造化（Ripple Carry Adder）
`alu_adder_nbit` で `alu_adder_1bit` を 16 個並べることで、下位ビットから上位ビットへとキャリー（桁上げ）が伝わっていく物理的な流れを視覚化します。

#### ③ フラグ生成ロジックの分離
特に複雑な **オーバーフロー（V）フラグ** の判定ロジックを `alu_arithmetic` 内に記述し、符号ビット（MSB）の変化を監視している様子を明確にします。また、**ゼロフラグ（Z）** は演算結果の全ビットを大きなNORゲート（`alu_flag_z_calc`）に入力する形で表現し、判定の仕組みを視覚化します。

### 4. 教育的な効果
合成後の回路図では、左側に入力データが入り、中央で複数の演算ブロック（Arithmetic, Logical, Shifter）に分かれ、右側の MUX で統合されて出力されるという、典型的な ALU の教科書的構造が現れます。これにより、命令ごとの「データの流れ」を追跡することが容易になります。
