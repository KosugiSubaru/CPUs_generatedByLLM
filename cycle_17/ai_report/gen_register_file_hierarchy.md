教育用シングルサイクルCPUにおいて、レジスタファイルは「データの保管場所」であり、命令がどのレジスタから値を読み、どこへ書き込むかを可視化する中心的なモジュールです。

16個の16bitレジスタを、単なるメモリ配列(`reg [15:0] rf [15:0]`)として記述するのではなく、**「16個の独立した箱（レジスタセル）」**と**「巨大なセレクタ（読み出しポート）」**として階層化することで、ハードウェアの実体（Dフリップフロップの集合体）を視覚的に理解できる構造を提案します。

### 1. 階層構造（Hierarchy）の定義

回路図で「16個の並列なレジスタ」と「2つの読み出し経路」が明確に見えるよう、以下のレベルで構成します。

*   **Level 0 (Primitive):** 最小単位（Enable付1bit DFF、1bit幅の16入力MUX）。
*   **Level 1 (Structural):** 16bitレジスタセル、および16bit幅の16入力セレクタ。
*   **Level 2 (Top/Logic):** これらを統合し、R0のゼロ固定や書き込みデコード論理を実装する層。

---

### 2. ファイル構成と役割

1ファイル1モジュールの原則に基づき、以下の5つのファイルで構成します。

| ファイル名（モジュール名） | 役割説明 | 階層レベル |
| :--- | :--- | :--- |
| `register_file_dff_en.v` | Enable信号付きの1bit D-フリップフロップ。レジスタの1ビット分を保持する。 | Level 0 |
| `register_file_cell_16bit.v` | `dff_en`を16個`generate`で並べたもの。一つの「レジスタ（R0〜R15）」を構成する。 | Level 1 |
| `register_file_mux16_1bit.v` | 16個のレジスタから、特定のレジスタの「特定の1ビット」を選択して出力する16入力1出力セレクタ。 | Level 0 |
| `register_file_mux16_16bit.v` | `mux16_1bit`を16個並べ、16個の16bit入力から1つの16bit出力を選択する巨大な読み出しポート。 | Level 1 |
| `register_file.v` | 16個の`cell_16bit`と、2つの`mux16_16bit`（2ポート読み出し）を結合するトップ。 | Level 2 |

---

### 3. パタン構造化と視覚化のポイント

#### 16個のレジスタの可視化
`register_file.v`（トップ）において、`register_file_cell_16bit` を `generate` 文で16回呼び出します。
論理合成後の回路図では、**「同じ形のレジスタブロックが16個縦に並ぶ」**様子が見て取れます。
*   ※ R0セルについては、書き込みEnableを常に無効、出力を常に0に固定する論理をトップで記述することで、「ゼロレジスタ」の特殊性を回路の形で示します。

#### 巨大な読み出しポート（MUXの階層化）
16入力のMUXを直接記述せず、1bit単位のセレクタ（`mux16_1bit`）として定義してから16bit分束ねることで、回路図上で**「16本のバスが2つの巨大な選択ブロックに入力されている」**様子が整理されて表示されます。これにより、`rs1`と`rs2`が独立したハードウェアであることを理解しやすくなります。

#### 書き込みデコード論理
どのレジスタに書き込むかを選択するデコーダは、トップモジュール内で「どのレジスタのEnable信号をHighにするか」というANDゲートの並びとして表現されます。これにより、「命令の `rd` フィールドがレジスタの書き込みスイッチを叩く」という概念が視覚化されます。

---

### 4. 教育的な付加価値
この構成により、学生は単なるプログラミングとしての「変数への代入」ではなく、**「クロック、データ、書き込み許可信号が物理的な記憶素子（DFF）に接続されている」**というコンピュータ・アーキテクチャの本質を学ぶことができます。
