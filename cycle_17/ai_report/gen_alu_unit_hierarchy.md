教育用シングルサイクルCPUにおいて、ALU（演算装置）は命令の実行段階の核となります。算術演算、論理演算、シフト演算がどのようにハードウェアで実現され、結果が選択されるかを視覚化するため、機能ごとの並列構造と、1ビット単位の積み上げ（ビットスライス）による階層化を提案します。

### 1. 階層構造（Hierarchy）の定義

回路図で「各演算が並列に計算され、最後に1つが選ばれる」様子が見えるよう、以下のレベルで構成します。

*   **Level 0 (Primitive):** 最小単位（1bitフルアダー、1bit論理スライス、1bit 8入力MUX）。
*   **Level 1 (Functional Block):** Level 0を`generate`文で連結した16bit演算器。
    *   Adder/Subtractor: 加減算器
    *   Logic Unit: 論理演算器（AND, OR, XOR, NOT）
    *   Shifter: シフト演算器（SRA, SLA）
*   **Level 2 (Management/Selection):** 最終的な演算結果を選択し、フラグを生成する層。
*   **Level 3 (Top):** これらを統合し、ALUユニット全体を構成する層。

### 2. ファイル構成と役割

1ファイル1モジュールの原則に基づき、以下の8つのファイルで構成します。

| ファイル名（モジュール名） | 役割説明 | 階層レベル |
| :--- | :--- | :--- |
| `alu_unit_full_adder.v` | 1bitのフルアダー。加減算の最小単位。 | Level 0 |
| `alu_unit_adder_16bit.v` | `full_adder`を16個連結。2の補数を利用した減算機能を含む16bit加減算器。 | Level 1 |
| `alu_unit_logic_slice.v` | 1bitのAND, OR, XOR, NOT演算をまとめて行う論理スライス。 | Level 0 |
| `alu_unit_logic_16bit.v` | `logic_slice`を16個並列に`generate`し、16bitの論理演算を行う。 | Level 1 |
| `alu_unit_shifter_16bit.v` | 指定されたシフト量に基づいてSRA（算術右シフト）とSLA（算術左シフト）を行う。 | Level 1 |
| `alu_unit_mux8_1bit.v` | 8つの入力信号から1つを選択する最小セレクタ。 | Level 0 |
| `alu_unit_mux8_16bit.v` | `mux8_1bit`を16個`generate`し、各演算結果から16bitの結果を選択する。 | Level 2 |
| `alu_unit_flags.v` | 最終結果と演算過程の信号からZ（ゼロ）, N（負）, V（溢れ）フラグを計算する。 | Level 2 |
| `alu_unit.v` | 各演算器を並列に動かし、結果選択とフラグ生成を統合するALUトップ。 | Level 3 |

### 3. パタン構造化と視覚化のポイント

#### 加減算の統合 (`adder_16bit`)
減算命令の際、第2入力のビットを反転させ、キャリーイン(Cin)に1を入力することで「2の補数」による加算として減算を実現します。回路図上では、**「減算の時だけBの入力にNOTゲートの列が挟まる」**様子が視覚化され、デジタル算術の基礎を学べます。

#### 論理演算の並列性 (`logic_16bit`)
AND, OR, XOR, NOTの4つを1つの「論理ブロック」として階層化します。各ビットにおいて、これらの演算が同時に行われていることが回路の形状から理解できます。

#### 演算結果の選択 (`mux8_16bit`)
本ISAのALU系命令に対応する8種類の結果（Add, Sub, And, Or, Xor, Not, SRA, SLA）を、1つの大きなセレクタに集約します。
回路図上では、**「全ての演算器が常に動いて計算結果を出しているが、命令(Opcode)によって1つの結果だけがrdへ流れていく」**というデータパスの概念を視覚的に表現します。

#### フラグ生成の独立 (`flags`)
フラグ計算を独立させることで、Z/N/Vフラグがどのように演算結果から導出されるかの論理を明確にします。特にV（オーバーフロー）フラグについては、入力と出力の符号ビットの関係がOR/ANDゲートで組まれている様子が可視化されます。

### 4. 教育的な付加価値
この構成により、学生は「ALU」という1つの大きな箱の中身が、実は「単純なビットごとの演算の積み重ね（パタン構造）」と「用途に応じた機能ブロックの並列配置（階層構造）」でできていることを、コードと回路図の両面から体感することができます。
