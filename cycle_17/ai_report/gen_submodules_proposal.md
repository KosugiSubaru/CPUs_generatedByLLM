教育用シングルサイクルCPUとして、ISAの各命令を直感的に理解できるように構成したサブモジュール案を提案します。

### 1. 提案するサブモジュール構成

CPU内部（`cpu`モジュール内）を以下の7つの主要モジュールに分割することを推奨します。

| モジュール名 | 役割 |
| :--- | :--- |
| **1. PC (Program Counter)** | 次に実行する命令のアドレスを保持・更新する。通常は+2するが、分岐・ジャンプ時はその先を指定する。 |
| **2. Decoder / Control Unit** | 命令(`i_instr`)を解析し、各部への制御信号（ALUの演算種別、レジスタ書き込み許可など）を生成する。 |
| **3. Register File** | 16個の16bitレジスタを管理。R0は常に0を出力する。2並列読み出し、1書き込みが可能。 |
| **4. Imm Gen (Immediate Generator)** | 命令内の即値フィールドを抽出し、16bitに符号拡張する。命令形式(R/I/J/B-type)に合わせて抽出位置を切り替える。 |
| **5. ALU (Arithmetic Logic Unit)** | 算術・論理演算、シフト演算を行う。演算結果とともに、フラグ（Z, N, V）を出力する。 |
| **6. Flag Register** | ALUが生成したフラグを保存する。条件分岐命令は、このレジスタに保持された「1つ前の命令のフラグ」を参照する。 |
| **7. Data Path Muxes** | ALUの入力（レジスタ値 vs 即値）や、レジスタへの書き込みデータ（ALU結果 vs メモリ値 vs PC+2）を切り替える。 |

---

### 2. CPUの動作フロー（相互作用の説明）

シングルサイクルCPUのため、以下の1から5の工程が1クロックサイクルの間に一気に行われます。

#### ① Fetch (命令取得)
*   **PC**が現在の命令アドレスを`o_addr_to_imem`へ出力し、命令メモリから命令(`i_instr`)を取得します。

#### ② Decode (解読) & Register Read (読み出し)
*   **Decoder**が命令を解析し、`rs1`, `rs2`, `rd`のアドレスを特定します。
*   **Register File**から`rs1`, `rs2`のデータが読み出されます。
*   同時に**Imm Gen**が、命令から即値を抽出し符号拡張して準備します。

#### ③ Execute (演算) & Flag Update (フラグ更新)
*   **ALU**が、レジスタから読み出した値（または即値）を用いて演算を行います。
*   演算結果に基づき、新しいフラグ(Z, N, V)が計算されます。
*   **【重要】** 本ISAの仕様に基づき、**Flag Register**は現在の演算結果を「次のクロック」のために保存し、分岐命令は「現在のレジスタに保存されている値（前の命令の結果）」を参照して分岐判断を行います。

#### ④ Memory Access (メモリ操作)
*   `load`/`store`命令の場合、ALUで計算されたアドレスを`o_addr_to_dmem`へ出力します。
*   `store`ならレジスタの値をメモリへ書き込み(`o_dmem_wen`をHighに)、`load`ならメモリからの値を`i_data_from_dmem`として受け取ります。

#### ⑤ Write Back (書き込み) & PC Update (次アドレス決定)
*   **Register File**の`rd`に対して、ALUの結果、メモリから読み出した値、または`jal`用の`PC+2`のいずれかを書き込みます。
*   **PC**は、通常は`PC+2`に更新されますが、分岐条件が成立した場合やジャンプ命令の場合は、計算された分岐先アドレスに更新されます。

### 3. 教育的な視点でのポイント

この構成をとることで、デバッグ用ポート(`o_debug_...`)と各モジュールの入出力を1対1で対応させやすくなります。

*   **視覚化:** 「今どの命令(Decoder)を実行していて、どのレジスタ(Register File)から値が出て、ALUでどう計算され、最終的にどこに書き戻されたか」というデータの流れ（データパス）を、モジュール境界ごとに明確に観測できるようになります。
*   **フラグの理解:** 「条件分岐が1クロック前の結果を見る」という仕様を、独立した**Flag Register**モジュールとして切り出すことで、回路上の「遅延（レジスタによる保持）」を学生が理解しやすくなります。
