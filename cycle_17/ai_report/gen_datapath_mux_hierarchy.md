教育用シングルサイクルCPUにおいて、`datapath_mux`（データパス・マルチプレクサ）は、「情報の交差点」の役割を果たします。命令によって、「レジスタの値を使うのか、即値を使うのか」「レジスタに書き戻すのは計算結果か、メモリの値か」といったデータの流れを切り替えます。

この交差点を、単なる`case`文ではなく、1ビット単位のパーツから組み上げる**ビットスライス構造**で階層化することで、データバスが物理的に切り替わる様子を視覚化します。

### 1. 階層構造（Hierarchy）の定義

論理合成後の回路図で、各セレクタが独立したブロックとして認識でき、かつその内部が規則的なビットの並びであることを示せるよう、以下のレベルで構成します。

*   **Level 0 (Primitive):** 最小単位（1bit 2入力MUX、1bit 4入力MUX）。
*   **Level 1 (Structural):** Level 0を`generate`文で連結した16bit幅のセレクタ。
*   **Level 2 (Top/Logic):** 実際のCPUのデータパス（ALU入力選択、レジスタ書き戻し選択）に合わせた統合層。

---

### 2. ファイル構成と役割

1ファイル1モジュールの原則に基づき、以下の5つのファイルで構成します。

| ファイル名（モジュール名） | 役割説明 | 階層レベル |
| :--- | :--- | :--- |
| `datapath_mux_2to1_1bit.v` | 1bitの2入力セレクタ。ALU入力Bの選択などの基礎。 | Level 0 |
| `datapath_mux_4to1_1bit.v` | 2入力MUXを組み合わせて作る1bitの4入力セレクタ。書き戻し選択の基礎。 | Level 0 |
| `datapath_mux_2to1_16bit.v` | `2to1_1bit`を16個並列に配置。16bit幅のデータ選択を行う。 | Level 1 |
| `datapath_mux_4to1_16bit.v` | `4to1_1bit`を16個並列に配置。16bit幅のデータ選択を行う。 | Level 1 |
| `datapath_mux.v` | CPU全体のデータパス上のMUXをまとめ、制御信号によって各経路を確立するトップ。 | Level 2 |

### 3. パタン構造化と視覚化のポイント

#### ALU入力選択の視覚化 (`2to1_16bit`)
`addi`や`load`命令では、ALUの第2入力に「レジスタ値」ではなく「即値」を流す必要があります。
回路図上では、**「rs2からのバスと、ImmGenからのバスが並んで走り、セレクタによって一方がALUの入力ピンへ接続される」**様子が視覚化されます。

#### レジスタ書き戻しデータの選択 (`4to1_16bit`)
本ISAでは、レジスタ(`rd`)に書き込むデータとして以下の4種類があります。
1.  **ALU結果:** 通常の演算
2.  **メモリ読出値:** `load`命令
3.  **PC+2:** `jal`, `jalr`命令の戻りアドレス
4.  **即値:** `loadi`命令
これを4入力の16bitセレクタとして階層化することで、**「CPU内の主要な4つのデータソースが1点に集まり、そこから1つだけがレジスタファイルへと戻っていく」**というデータパスの「還流」を視覚的に表現します。

#### パタン構造化によるバスの表現
全ての16bitセレクタを `generate` 文による1bitパーツの配列として実装します。これにより、回路図の拡大時に、**「16個の同じスイッチがずらりと並んで、巨大なバスを制御している」**というデジタル回路の規則性を直感的に理解できるようになります。

### 4. 教育的な付加価値
この構成により、学生は以下のステップでデータパスを学習できます。
1.  **接続の切り替え:** CPUは配線をつなぎ替えるのではなく、セレクタによって信号を通したり遮断したりして動作を切り替えていることを知る。
2.  **制御信号の役割:** デコーダから出る1bitや2bitの信号が、いかにして16bitという太い情報の流れを統制しているかを理解する。
3.  **データパスの全景:** `datapath_mux` モジュールをデバッグポートで観測することで、命令ごとに「今、どのデータが選ばれて計算されているか」を追跡可能にする。
