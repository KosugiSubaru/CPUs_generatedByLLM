教育用シングルサイクルCPUとして、ISAの各命令を効率的かつ視覚的に分かりやすく実行するために必要なサブモジュールと、それらの相互作用を提案します。

---

### 1. 提案するサブモジュール一覧

CPU内部を以下の7つの主要なモジュールに分割することを提案します。

| モジュール名 | 役割 |
| :--- | :--- |
| **1. PC (Program Counter)** | 次に実行する命令のメモリ番地（16bit）を保持するレジスタ。 |
| **2. Control Unit** | 命令（Opcode）をデコードし、各部への選択信号（セレクタ切り替えや書き込み許可）を生成する「司令塔」。 |
| **3. Register File** | 16個の16bitレジスタを格納。R0は常に0を出力。2ポート読み出し、1ポート書き込みが可能。 |
| **4. Imm Gen (Immediate Generator)** | 命令内のビットフィールドから即値を取り出し、符号拡張を行って16bitデータに整形する。 |
| **5. ALU (Arithmetic Logic Unit)** | 加減算、論理演算、シフト演算を実行。結果とともにフラグ（Z, N, V）を出力する。 |
| **6. Flag Register** | ALUが出力したフラグを1クロック保持する。※ISAの「1クロック前のフラグを参照する」という仕様に必要。 |
| **7. Next PC Logic (Branch Control)** | 分岐条件の判定と、次のPC値（PC+2、PC+imm、rs1+imm）の算出を行う。 |

---

### 2. 各モジュールの詳細と役割

#### ① PC (Program Counter)
- **役割**: 現在の実行アドレスを保持。
- **動作**: クロックの立ち上がりに同期して、`Next PC Logic` から送られてくる次のアドレスに更新される。

#### ② Control Unit
- **役割**: 命令の[3:0]ビット（Opcode）を見て、以下の信号を制御する。
    - ALUにどの演算をさせるか（ALUOp）
    - レジスタファイルに書き込むか（RegWrite）
    - データメモリに書き込むか（MemWrite）
    - ALUの入力に即値を使うかレジスタ値を使うか（ALUSrc）
    - 書き戻しデータとして ALU / メモリ / PC+2 のどれを選ぶか（ResultSrc）

#### ③ Register File
- **役割**: プログラムで使用する一時データを保持。
- **動作**: `rs1`, `rs2` のアドレスに基づき2つのデータを非同期で出力。`rd` への書き込みは、`RegWrite` が有効なとき、クロック立ち上がりで行われる。

#### ④ Imm Gen (Immediate Generator)
- **役割**: 命令形式（R/I/J/B-type）に合わせて即値を抽出。
- **動作**: 4bit, 8bit, 12bitの各即値を16bitに符号拡張（Sign-extend）する。

#### ⑤ ALU (Arithmetic Logic Unit)
- **役割**: 算術演算と論理演算の実体。
- **動作**: `rs1` の値と、`rs2` または `imm` を演算。
- **フラグ出力**: 計算結果に基づき、Z（ゼロ）、N（負）、V（オーバーフロー）を即座に出力する。

#### ⑥ Flag Register
- **役割**: 比較結果の保存。
- **動作**: ALUが出力した Z, N, V フラグをクロック同期で保存。分岐命令（`blt`, `bz`）は、**このレジスタに保存されている値**を見て分岐するか決定する。

#### ⑦ Next PC Logic
- **役割**: 次のPCの計算。
- **動作**:
    - 通常：`PC + 2`
    - 分岐成功時（bz等）：`PC + imm`
    - ジャンプ（jal）：`PC + imm`
    - レジスタジャンプ（jalr）：`rs1 + imm`

---

### 3. CPUの動作フロー（相互作用の説明）

シングルサイクルであるため、1クロックの間に以下の流れが同時並行（組み合わせ回路の伝搬）で行われます。

1.  **Instruction Fetch**:
    `PC` が示すアドレスの命令を `imem` から読み出す。
2.  **Decode & Register Read**:
    `Control Unit` が命令を解析。同時に `Register File` から `rs1`, `rs2` のデータを読み出す。`Imm Gen` が即値を生成する。
3.  **Execute**:
    `ALU` が演算（加算や論理演算）を行う。ここで出たフラグは `Flag Register` へ送られる（実際に保存されるのは次のクロック）。
4.  **Memory Access**:
    `load` 命令なら `dmem` からデータを読み出し、`store` 命令なら `rs2` のデータを `dmem` に書き込む。
5.  **Write Back**:
    `ALU` の計算結果、または `dmem` から読み出したデータを `rd` レジスタに書き込む。
6.  **Next PC Determination**:
    `Flag Register`（前回の命令の結果）と命令内容に基づき、`Next PC Logic` が次サイクルの `PC` を決定する。

### 教育的視点でのポイント
この構成にすることで、「命令のビットが物理的にどこのスイッチ（セレクタ）を動かしているか」が可視化されます。特に、**Flag Register** を独立させることで、「比較命令の次のサイクルでしか分岐できない」というISAの制約が、回路上の「フリップフロップによる遅延」として直感的に理解できるようになります。
