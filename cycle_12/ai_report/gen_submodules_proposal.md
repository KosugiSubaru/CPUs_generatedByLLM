教育用シングルサイクルCPUの設計として、ISAの定義と提供されたシステム構成に基づき、`cpu`モジュール内部を構成するサブモジュールを以下に提案します。

---

### 1. 提案するサブモジュール構成

CPU内部を以下の7つの主要モジュールに分割することを提案します。

#### ① Program Counter (PC)
- **役割**: 次に実行する命令のメモリ番地（`w_addr_to_imem`）を保持するレジスタです。
- **動作**: 毎クロック、通常は `PC + 2` を出力しますが、分岐（Branch）やジャンプ（JAL/JALR）時には計算された飛び先の番地へと更新されます。

#### ② Instruction Decoder (ID / Control Unit)
- **役割**: 命令（`i_instr`）を解析し、各モジュールへの制御信号を生成する「司令塔」です。
- **動作**: 命令の3:0ビット（opcode）を見て、ALUの演算種類、レジスタ書き込み許可（RegWrite）、メモリ書き込み許可（`o_dmem_wen`）、データ選択セレクタの切り替え信号などを出力します。

#### ③ Register File (RF)
- **役割**: 16ビット×16本のレジスタ群です。
- **動作**: 命令に含まれる `rs1`, `rs2` アドレスに基づきデータを出力します。また、書き込み有効時に `rd` アドレスへ計算結果を保存します。
- **特殊仕様**: ISAの定義通り、レジスタ番号0は常に値`0`を返し、書き込みを無視する論理を内蔵します。

#### ④ ALU (Arithmetic Logic Unit)
- **役割**: 算術演算、論理演算、シフト演算を行います。
- **動作**: `rs1` の値と、`rs2` または即値（Immediate）を入力として演算を行い、結果とフラグ（Z, N, V）を出力します。

#### ⑤ Flag Register
- **役割**: ALUが計算したフラグ（Z, N, V）を1クロック保持するレジスタです。
- **動作**: ISAの「条件分岐は1クロック前のフラグを参照する」という定義に基づき、現在の命令がALU命令であればフラグを更新し、次サイクルの分岐命令がその値を使用できるようにします。

#### ⑥ Immediate Generator (ImmGen)
- **役割**: 命令内の異なる位置にある即値ビットを抽出し、16ビットに符号拡張します。
- **動作**: 命令形式（I-type, Jump用, Branch用など）に応じて、ビットを並び替えて正しい数値（符号付き整数）に整形します。

#### ⑦ Data Path Multiplexers (MUX)
- **役割**: データの流れを切り替えるスイッチ群です。
- **動作**: 
    - ALUの入力として「レジスタ値」か「即値」かを選択。
    - レジスタへの書き込みデータとして「ALU結果」「メモリからの読み出し値」「PC+2（JAL用）」かを選択。

---

### 2. CPUの動作プロセス（相互作用の簡潔な説明）

シングルサイクルCPUとして、1クロックの間に以下の流れが同時に（組み合わせ回路として）発生します。

1.  **命令フェッチ**: `PC` が指すアドレスの命令が `imem` から届きます。
2.  **デコードと読み出し**: `Instruction Decoder` が命令を解析し、同時に `Register File` からソースレジスタ（`rs1`, `rs2`）の値が読み出されます。
3.  **実行 (ALU)**: `ALU` が、読み出されたレジスタ値や `ImmGen` で生成された即値を使って計算します。この時、フラグ（Z, N, V）も計算されます。
4.  **メモリアクセス**: 
    - `load` 命令なら、ALUの計算結果をアドレスとして `dmem` からデータを読み出します。
    - `store` 命令なら、`o_dmem_wen` を有効にしてデータを `dmem` へ書き込みます。
5.  **ライトバック**: レジスタ書き込みが有効な命令（add, load, loadi等）の場合、演算結果やメモリからのデータを `Register File` の `rd` に保存します。
6.  **次PCの決定**: 
    - 分岐命令（`blt`, `bz`）の場合、**Flag Registerに保存されている1つ前のフラグ値**を確認し、条件を満たせば `PC + imm`、そうでなければ `PC + 2` を次のPC値として準備します。
    - クロックの立ち上がりで、`PC` と `Flag Register` が同時に更新されます。

### 教育的視点でのポイント
この構成にすることで、**「命令のどのビットが、どの回路のスイッチを動かしているか」**（例：opcode[3:0]がALUの機能選択に直結している点など）を視覚的に分かりやすく図解・実装することが可能になります。
