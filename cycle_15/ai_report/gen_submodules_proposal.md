教育用シングルサイクルCPUとして、ISAの定義に基づいた構成案を提案します。

このCPUは「1つの命令を1クロックで実行する」という基本原則に基づきつつ、**「条件分岐は1クロック前のフラグを参照する」**という仕様を満たすために、フラグを保存するレジスタ（Flag Register）を明示的に配置する構成をとります。

---

### 1. 提案するサブモジュール構成

CPU（`u_cpu`）の内部を以下の7つのサブモジュールに分割します。

| モジュール名 | 役割 |
| :--- | :--- |
| **1. Program Counter (PC)** | 次に実行する命令のアドレスを保持するレジスタ。 |
| **2. Control Unit** | 命令の`opcode`（下位4bit等）をデコードし、ALUの演算種別や各マルチプレクサ、レジスタ書き込み有効化などの制御信号を生成する。 |
| **3. Register File** | 16ビット×16本のレジスタ群。2つの読み出しポートと1つの書き込みポートを持つ。R0は常に0を出力し、書き込みを無視する。 |
| **4. Imm Gen (Immediate Generator)** | 命令内のビットフィールドから即値を取り出し、16ビットに符号拡張する。 |
| **5. ALU (Arithmetic Logic Unit)** | 算術論理演算（加減算、論理演算、シフト）を実行。演算結果とともに、フラグ（Z, N, V）を出力する。 |
| **6. Flag Register** | ALUが出力したフラグ（Z, N, V）をクロック立ち上がりで保存する。分岐命令はこのレジスタの値を参照する。 |
| **7. Next PC Logic** | 通常のPC+2、分岐（PC+imm）、ジャンプ（PC+imm または rs1+imm）のいずれを次のアドレスにするかを判定する。 |

---

### 2. モジュール間の相互作用と動作フロー

シングルサイクル内でのデータの流れは以下の通りです。

#### ① Fetch (命令取得)
- `PC`が出力するアドレス（`o_addr_to_imem`）に基づき、命令メモリから命令（`i_instr`）が供給されます。
  - ※提供された`imem`コードが同期読み出し（1クロック遅れ）のため、実際には「現在のPCに対する命令」が供給されている状態からサイクルが開始されます。

#### ② Decode & Register Read (解読とレジスタ読み出し)
- `Control Unit`が命令の下位4ビット（opcode）を見て、以下の制御を行います。
  - レジスタの書き込み有無（`reg_wen`）
  - データメモリの書き込み有無（`o_dmem_wen`）
  - ALUに入力するデータの選択（レジスタ値か即値か）
- `Register File`は、命令フィールド（rd, rs1, rs2）に従ってデータを出力します。
- `Imm Gen`は、命令の形式に合わせて即値を生成します。

#### ③ Execute & Flag Update (演算とフラグ更新)
- `ALU`は、選択された入力（rs1, rs2/imm）に対して演算を行い、結果を出力します。
- **重要（フラグ仕様）**: ALUが算出した最新のフラグ（Z, N, V）は`Flag Register`へ送られ、**次のクロック立ち上がり**で保存されます。

#### ④ Memory Access (データメモリ)
- `load`命令の場合：ALUの結果（rs1+imm）をアドレスとして、`dmem`からデータを読み出します。
- `store`命令の場合：ALUの結果をアドレスとして、`rs2`の値を`dmem`に書き込みます（`o_dmem_wen`を有効化）。

#### ⑤ Write Back (レジスタ書き戻し)
- `Register File`の書き込みポートに対し、以下のいずれかを選択して書き込みます。
  - ALUの演算結果（R-type, addi, loadi）
  - メモリから読み出したデータ（load）
  - PC + 2（jal, jalr）

#### ⑥ Next PC Calculation (次アドレス決定)
- `Next PC Logic`が、現在の命令が分岐・ジャンプ命令かを判定します。
- **条件分岐（blt, bz）の場合**: **`Flag Register`（前サイクルの結果）**を参照し、条件を満たせば `PC + imm`、満たさなければ `PC + 2` を選択します。
- **ジャンプ（jal, jalr）の場合**: `PC + imm` または `rs1 + imm` を強制的に選択します。

---

### 3. この構成のポイント（教育的観点）

1.  **フラグの可視化**: 
    「1クロック前のフラグを参照する」という仕様により、ALUの計算結果が即座に分岐に影響せず、ワンテンポ遅れて反映される様子を、`Flag Register`という独立したモジュールを通じて直感的に理解できます。
2.  **データパスの分離**: 
    演算（ALU）、メモリアクセス（dmem）、制御フロー（Next PC Logic）を分離することで、CPUの3大要素（処理、記憶、制御）の役割分担が明確になります。
3.  **ゼロレジスタの扱い**: 
    `Register File`内部でR0をハードウェア的に0固定にすることで、ソフトウェア（ISA）の制約がどのようにハードウェアで実現されるかを学べます。
