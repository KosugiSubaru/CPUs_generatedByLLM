サブモジュール `writeback_mux` について、レジスタファイルに書き戻すデータの選択（ALU演算結果、メモリ読出データ、または戻りアドレス）を構造化し、ビットスライス構造によって視覚化する階層設計を提案します。

この設計では、16ビットのデータパスを1ビットごとのセレクタに分解し、それを `generate` 文で並列展開することで、デジタル回路における「ビット並列処理」の概念を視覚的に理解できるようにします。

---

### 1. 提案する階層構造一覧

| 階層レベル | ファイル名 | 役割 |
| :--- | :--- | :--- |
| **Level 0** | `writeback_mux.v` | 最上位。CPU内部の各ユニット（ALU, Mem, PC）からの信号を統合する。 |
| **Level 1** | `writeback_mux_16bit_4to1.v` | 16ビット幅の4入力セレクタ。1ビットセレクタを16個並列配置する。 |
| **Level 2** | `writeback_mux_1bit_4to1.v` | 1ビット単位の4入力セレクタ。2入力セレクタを組み合わせて構成。 |
| **Level 3** | `writeback_mux_1bit_2to1.v` | セレクタの最小構成単位。1ビットの2入力セレクタ。 |

---

### 2. 各モジュールの役割と動作説明

#### 相互作用と動作フロー
1.  **入力供給:** ALUでの計算結果（Arithmetic/Logic）、データメモリからの読出値（Load）、および `next_pc_logic` からの `PC + 2`（JAL/JALRの戻りアドレス）の3つのデータがこのモジュールに集約されます。
2.  **制御信号:** `Control Unit` から出力される `reg_wb_sel`（2ビット）に基づき、どのデータをレジスタファイルに書き戻すかを決定します。
3.  **書き戻し:** 選択された16ビットデータは `Register File` の書き込みデータ入力（`wd`）へと送られ、クロックに同期して `rd` レジスタに格納されます。

#### 各ファイルの役割
- **`writeback_mux.v`**
    - **役割:** インターフェースの定義。バスの接続を行い、教育用CPUのデータパスにおける「合流地点」としての役割を回路図上で明示します。
- **`writeback_mux_16bit_4to1.v`**
    - **構造化:** **`generate` 文**を使用して、`writeback_mux_1bit_4to1` を16個並列にインスタンス化します。
    - **教育的意図:** 16ビットのデータ選択が、実は1ビットずつの独立した選択処理の集まりであることを視覚化します。
- **`writeback_mux_1bit_4to1.v`**
    - **構造化:** `writeback_mux_1bit_2to1` を3つツリー状に組み合わせて構築します。
    - **教育的意図:** 複雑な多入力選択が、単純な2択の積み重ね（デシジョンツリー）で実現できることを示します。
- **`writeback_mux_1bit_2to1.v`**
    - **役割:** 最小単位の選択論理（基本ゲートの組み合わせ、または三項演算子による記述）。

---

### 3. パタン構造化（generate文）の適用イメージ

`writeback_mux_16bit_4to1.v` 内で、ビットごとに構造を繰り返すことで、合成後の回路図では16個のセレクタブロックが整然と並ぶ様子が確認できます。

```verilog
// writeback_mux_16bit_4to1.v 内のイメージ
genvar i;
generate
    for (i = 0; i < 16; i = i + 1) begin : gen_wb_slices
        writeback_mux_1bit_4to1 u_slice (
            .i_sel (i_sel),
            .i_d0  (i_alu_res[i]),
            .i_d1  (i_mem_data[i]),
            .i_d2  (i_pc_plus_2[i]),
            .i_d3  (1'b0), // 予約/未使用
            .o_q   (o_wb_data[i])
        );
    end
endgenerate
```

### この構成のメリット
1.  **データパスの明瞭化:** 合成後の回路図で、データがどこから来てどこへ行くのかが、太いバス（16bit）とそれを処理する個別のビットセルとして明確に分離されます。
2.  **階層的な理解:** 「1ビットの選択」から「16ビットのバス選択」へ、そして「CPUの書き戻し制御」へと、ボトムアップに設計が積み上がる過程を視覚的に追跡できます。
3.  **ISAとの対応:** `load`（メモリから）、`add`（ALUから）、`jal`（PC+2から）というISAの挙動が、このMUXの入力の切り替えに対応していることが直感的に理解できます。
