サブモジュール `alu` について、各演算器の独立性と、1ビット単位から構築する階層構造を重視した設計を提案します。

論理合成後の回路図において、「算術演算」「論理演算」「シフト演算」が独立したブロックとして並び、それらが最終的に大きなセレクタで集約される様子を視覚化します。

---

### 1. 階層構造一覧

| 階層レベル | ファイル名 | 役割 |
| :--- | :--- | :--- |
| **Level 0** | `alu.v` | 最上位。各演算ユニットを統合し、最終結果とフラグを出力する。 |
| **Level 1** | `alu_arithmetic_unit.v` | 加減算を担当。内部で16bit加算器を使用する。 |
| **Level 1** | `alu_logic_unit.v` | ビットごとの論理演算（AND, OR, XOR, NOT）を担当。 |
| **Level 1** | `alu_shift_unit.v` | 算術シフト（SRA, SLA）を担当。バレルシフタ構造を持つ。 |
| **Level 1** | `alu_flag_logic.v` | 演算結果からステータスフラグ（Z, N, V）を算出する。 |
| **Level 1** | `alu_mux_8to1_16bit.v` | 8つの演算結果候補から、Opcodeに応じて1つを選択する。 |
| **Level 2** | `alu_adder_16bit.v` | 16ビット構造化加算器。 |
| **Level 2** | `alu_mux_2to1_16bit.v` | 16ビット2入力セレクタ（8to1 MUXやシフタの構成要素）。 |
| **Level 3** | `alu_full_adder.v` | 1ビット全加算器。 |
| **Level 4** | `alu_half_adder.v` | 1ビット半加算器。 |

---

### 2. 各モジュールの役割と階層の詳細説明

#### Level 0: 統合
- **`alu.v`**
  - **役割:** 制御信号（opcode）に従い、各演算ユニットへデータを分配し、最終的な出力をまとめます。
  - **視覚的特徴:** 合成図の中央に演算ブロックが並び、右側にフラグ生成と結果選択MUXが配置される「プロセッサの心臓部」らしい構造になります。

#### Level 1: 演算機能ブロック
- **`alu_arithmetic_unit.v`**
  - **内容:** 加算と減算（Bの補数を利用）を切り替えます。
  - **教育的意図:** 減算が「Bを反転して+1加算する」というデジタル回路の基本原則を視覚化します。
- **`alu_logic_unit.v`**
  - **内容:** `generate` 文を用いて、ビットごとの論理ゲートを16個並列化します。
- **`alu_shift_unit.v`**
  - **内容:** 算術シフトを実行します。
  - **パタン構造:** `alu_mux_2to1_16bit` を階層的に配置（1, 2, 4, 8ビットシフトのステージ）することで、1クロックで多ビットシフトを行うバレルシフタの仕組みを視覚化します。
- **`alu_flag_logic.v`**
  - **内容:** ISA定義にあるオーバーフロー（V）の複雑な論理式や、ゼロ判定（Z）、符号判定（N）をまとめます。

#### Level 2, 3 & 4: 構造化加算器 (Adder Hierarchy)
- **`alu_adder_16bit.v`**
  - **構造:** `generate` 文を用いて、`alu_full_adder` を16個直列に接続（リプルキャリー構造）します。
- **`alu_full_adder.v`**
  - **構造:** 2つの `alu_half_adder` とORゲートで構成。
- **`alu_half_adder.v`**
  - **内容:** 加算の最小単位。

---

### 3. パタン構造化（generate文）の適用イメージ

例えば、論理演算ユニット（`alu_logic_unit.v`）では以下のようにビットごとの独立性を強調します。

```verilog
// alu_logic_unit.v 内の構造化イメージ
genvar i;
generate
    for (i = 0; i < 16; i = i + 1) begin : gen_bit_logic
        // ビットごとに論理ゲートが並列配置される様子を回路図で確認可能
        assign w_and_out[i] = i_a[i] & i_b[i];
        assign w_or_out[i]  = i_a[i] | i_b[i];
        assign w_xor_out[i] = i_a[i] ^ i_b[i];
        assign w_not_out[i] = ~i_a[i];
    end
endgenerate
```

### この構成のメリット
1.  **演算パスの分離:** 「今、加算をしているのかシフトをしているのか」がデータフローとして回路図上で追いやすくなります。
2.  **フラグ生成の明確化:** オーバーフロー判定（V）が算術演算ユニットの結果にのみ依存する様子などが、配線の繋がりから理解できます。
3.  **シフタの構造教育:** 2入力セレクタの組み合わせでバレルシフタを構築することで、複雑に見える多ビットシフトが実は単純な選択回路の積み重ねであることを示せます。
