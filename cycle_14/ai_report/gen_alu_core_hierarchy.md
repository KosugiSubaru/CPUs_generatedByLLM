教育用シングルサイクルCPUにおいて、**alu_core**（算術論理演算ユニット）は計算の核心部です。ISAで定義された「算術・論理・シフト」の各演算と「フラグ生成（Z, N, V）」の対応を視覚化するため、演算カテゴリごとにブロックを分割し、1ビット単位の処理を16ビットへ拡張する階層構造を提案します。

---

### 1. 階層構造の概要

| 階層レベル | モジュール名 | 役割 | パタン構造化の適用 |
| :--- | :--- | :--- | :--- |
| **L0 (ユニット)** | `alu_core` | ALUの最上位。各演算ブロックとフラグ生成、結果選択を統合。 | 構成要素の接続 |
| **L1 (機能ブロック)** | `alu_core_arithmetic_16bit` | 加減算およびオーバーフロー（V）判定を行う。 | `generate`によるフルアダーの連結 |
| | `alu_core_logical_16bit` | ビットごとの論理演算（AND/OR/XOR/NOT）を行う。 | `generate`による1bit論理スライスの並列化 |
| | `alu_core_shifter_16bit` | 算術シフト（SRA/SLA）を行う。 | 段階的なマルチプレクサ構造 |
| | `alu_core_res_mux_16bit` | 各演算結果から1つを選択する。 | `generate`による1bit幅セレクタの並列化 |
| **L2 (プリミティブ)** | `alu_core_full_adder` | 1ビット全加算器。 | 基本論理ゲート |
| | `alu_core_bit_logic` | 1ビット単位の論理ゲート集合体。 | 基本論理ゲート |
| | `alu_core_mux16_1bit` | 16個の1bit入力から1つを選ぶセレクタ（Opcode対応）。 | 基本論理ゲート |

---

### 2. 各ファイルの役割と階層レベルの説明

#### 【L2: プリミティブ層】
演算の最小構成要素です。回路図上で最小の「部品」として表示されます。

1.  **alu_core_full_adder.v**
    *   **役割:** 1ビットの加算とキャリー出力を担当。
2.  **alu_core_bit_logic.v**
    *   **役割:** 2入力に対するAND, OR, XOR, NOT演算を同時に行い、複数の線（Bus）として出力。
3.  **alu_core_mux16_1bit.v**
    *   **役割:** 16種類の演算候補（Opcodeの4bitに対応）から、1ビット分の計算結果を選択。

#### 【L1: 機能ブロック層】
プリミティブを組み合わせ、ISAの演算命令を具体化します。

4.  **alu_core_arithmetic_16bit.v**
    *   **役割:** `add`, `sub` 命令に対応。
    *   **構造:** `generate` 文で `alu_core_full_adder` を16個接続。減算時は入力を反転してキャリー入力を `1` にする（2の補数）。
    *   **フラグ:** 加減算時のみ有効なオーバーフロー（V）信号を生成。
5.  **alu_core_logical_16bit.v**
    *   **役割:** `and`, `or`, `xor`, `not` 命令に対応。
    *   **構造:** `generate` 文で `alu_core_bit_logic` を16個並列配置。
6.  **alu_core_shifter_16bit.v**
    *   **役割:** `sra`, `sla` 命令に対応。
    *   **構造:** バレルシフタ形式（8, 4, 2, 1ビットの選択器の連鎖）で視覚化し、算術シフト時の符号ビット維持を明示。
7.  **alu_core_res_mux_16bit.v**
    *   **役割:** Opcodeに従い、算術・論理・シフト・即値パス（loadi等）のいずれかの結果を出力へ繋ぐ。
    *   **構造:** `generate` 文で `alu_core_mux16_1bit` を16個並列化。

#### 【L0: ユニット層（最上位）】
外部（CPUデータパス）とのインターフェースおよびフラグ確定を行います。

8.  **alu_core.v**
    *   **役割:** 演算結果の確定と、共通フラグ（Z: ゼロ、N: 負）の生成。
    *   **視覚化の利点:** 「演算結果が決まった後にフラグが決定される」という依存関係が回路図上で明確になります。

---

### 3. 提案するサブモジュールリスト

submodules:
alu_core: ALUの最上位。各演算結果の選択と、共通ステータスフラグ（Z, N）の生成を行う。
alu_core_arithmetic_16bit: 加算・減算を実行し、算術演算特有のフラグ（V）を生成する。
alu_core_logical_16bit: ビットごとの論理演算（AND, OR, XOR, NOT）を並列実行する。
alu_core_shifter_16bit: 算術右シフトおよび算術左シフトを、シフト量（rs2の下位4bit）に基づいて実行する。
alu_core_res_mux_16bit: Opcodeに基づき、複数の演算器の中から1つの最終結果を選択する。
alu_core_full_adder: 最小単位の1bit全加算器。
alu_core_bit_logic: 最小単位の1bit論理演算スライス。
alu_core_mux16_1bit: 16個の1bit入力からOpcodeの4bit値に基づいて1つを選択する最小セレクタ。
