教育用シングルサイクルCPUにおいて、PC（プログラムカウンタ）周りは「命令の流れ」を制御する核心部です。ISAの定義（`pc+2`, `pc+imm`, `rs1+imm`）に基づき、**pc_reg** モジュールを以下の3つの階層レベルに分割・構造化することを提案します。

---

### 1. 階層構造の概要

| 階層レベル | モジュール名 | 役割 | パタン構造化の適用 |
| :--- | :--- | :--- | :--- |
| **L0 (ユニット)** | `pc_reg_unit` | PC周りの最上位。加算器、MUX、レジスタを結合 | 構成要素の接続 |
| **L1 (機能ブロック)** | `pc_reg_dff_16bit` | 16bitの状態保持レジスタ | `generate`による1bit DFFの並列化 |
| | `pc_reg_adder_16bit` | 16bit加算器（PC+2やオフセット計算用） | `generate`によるフルアダーの連結 |
| | `pc_reg_mux4_16bit` | 4入力16bitセレクタ（次アドレス選択用） | `generate`による1bit MUXの並列化 |
| **L2 (プリミティブ)** | `pc_reg_dff_1bit` | 1ビットD-フリップフロップ | 基本論理ゲート/FF |
| | `pc_reg_full_adder` | 1ビットフルアダー | 基本論理ゲート |
| | `pc_reg_mux4_1bit` | 4入力1ビットセレクタ | 基本論理ゲート |

---

### 2. 各ファイルの役割と構成

#### 【L2: プリミティブ層】
最小単位の論理を定義します。これらは視覚化された際に最も小さな「部品」として現れます。

1.  **pc_reg_full_adder.v**
    *   **役割:** 1ビットの全加算器。A, B, CarryInを入力し、SumとCarryOutを出力します。
2.  **pc_reg_mux4_1bit.v**
    *   **役割:** 4つの入力から1つを選択する1ビットセレクタ。2ビットの選択信号（sel）を使用します。
3.  **pc_reg_dff_1bit.v**
    *   **役割:** 非同期リセット付きの1ビットDフリップフロップ。

#### 【L1: 機能ブロック層】
`generate` 文を使用して、L2のプリミティブを16ビット幅に拡張します。

4.  **pc_reg_adder_16bit.v**
    *   **役割:** 16ビットのリップルキャリー加算器。
    *   **構造:** `generate` 文を用いて `pc_reg_full_adder` を16個インスタンス化し、キャリー信号を連鎖させます。これにより、回路図上で「加算の構造」が視覚化されます。
5.  **pc_reg_mux4_16bit.v**
    *   **役割:** 16ビット幅の4入力セレクタ。
    *   **構造:** `generate` 文を用いて `pc_reg_mux4_1bit` を16個並列にインスタンス化します。
6.  **pc_reg_dff_16bit.v**
    *   **役割:** 16ビットのレジスタ本体。
    *   **構造:** `generate` 文を用いて `pc_reg_dff_1bit` を16個インスタンス化します。

#### 【L0: ユニット層（最上位）】
ISAの仕様に基づき、次のPCアドレスを決定する論理を組み立てます。

7.  **pc_reg_unit.v**
    *   **役割:** PCユニットの統合。外部（imem）へ現在のアドレスを出力し、次のサイクルでのアドレスを確定させます。
    *   **内部構成:**
        *   `pc_reg_adder_16bit` (インスタンスA): `PC + 2` を計算（通常増分）。
        *   `pc_reg_adder_16bit` (インスタンスB): `PC + imm` を計算（条件分岐/JAL用）。
        *   `pc_reg_adder_16bit` (インスタンスC): `rs1 + imm` を計算（JALR用）。
        *   `pc_reg_mux4_16bit`: 制御信号に基づき、上記3つの計算結果または現在のPC（ホールド用）から次アドレスを選択。
        *   `pc_reg_dff_16bit`: 選択された次アドレスをクロック同期で保存。

---

### 3. ISAとの対応関係（視覚化のポイント）

この階層化により、論理合成後のツールで以下のような対応が視覚的に理解できるようになります。

*   **PC+2の計算:** 回路図上に固定値 `2` が入力された加算器ブロックが見えるため、命令長が16ビット（2バイト）であることが直感的にわかります。
*   **分岐アドレスの計算:** `pc_reg_unit` 内で `PC` と `imm` が加算器に入力されている様子が見え、相対ジャンプの仕組みが理解できます。
*   **ジャンプの選択:** `mux4` を通じて「通常時はPC+2、条件一致時はPC+imm、JALR時はrs1+imm」というアドレス選択の優先順位が回路の結線として示されます。

### サブモジュールリスト（最終確認）

submodules:
pc_reg_unit: PCの保持と次アドレス計算を統合する最上位ユニット
pc_reg_dff_16bit: 16bitの状態保持レジスタ（1bit DFF x16）
pc_reg_adder_16bit: 16bit加算器（Full Adder x16）
pc_reg_mux4_16bit: 16bit 4入力セレクタ（1bit Mux4 x16）
pc_reg_dff_1bit: 基本単位となる1bit Dフリップフロップ
pc_reg_full_adder: 基本単位となる1bit全加算器
pc_reg_mux4_1bit: 基本単位となる1bit 4入力セレクタ
