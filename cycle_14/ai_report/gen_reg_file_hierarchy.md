教育用シングルサイクルCPUにおいて、**reg_file**（レジスタファイル）は「データの保管庫」であり、同時に複数の場所から読み出し、一箇所へ書き込むという複雑な配線構造を持ちます。

ISAの「16個のレジスタ」「ゼロレジスタの固定」「2ポート読み出し」を視覚化するため、以下の階層構造を提案します。

---

### 1. 階層構造の概要

| 階層レベル | モジュール名 | 役割 | パタン構造化の適用 |
| :--- | :--- | :--- | :--- |
| **L0 (ユニット)** | `reg_file` | レジスタファイルの最上位。バンク、デコーダ、読出ポートを統合。 | 構成要素の接続 |
| **L1 (機能ブロック)** | `reg_file_bank_16` | 16個の16bitレジスタ集合体。**レジスタ0の固定**をここで実現。 | `generate`によるセル配置 |
| | `reg_file_read_port_16bit` | 16個の入力から1つを選ぶ16bit幅のセレクタ。 | `generate`による1bit幅セレクタの並列化 |
| | `reg_file_decoder_4to16` | 4bitのアドレスを16本の書き込み許可信号に変換。 | 論理ゲートによるデコード |
| **L2 (プリミティブ)** | `reg_file_cell_16bit` | 1ビット単位ではなく、**「1つのレジスタ（16bit）」**としての記憶単位。 | 16bitフリップフロップ |
| | `reg_file_mux_16to1_1bit` | 16個の1bit入力から1つを選ぶ、読出ポートの最小単位。 | セレクタ論理 |

---

### 2. 各ファイルの役割と階層レベルの説明

#### 【L2: プリミティブ層】
回路図上で「1つの箱」として認識される最小の構成要素です。

1.  **reg_file_cell_16bit.v**
    *   **役割:** 書き込み有効信号（EN）付きの16bitレジスタ。
    *   **視覚化の利点:** 回路図上で、データ幅16の太い線が入出力される1個の記憶素子として表現されます。
2.  **reg_file_mux_16to1_1bit.v**
    *   **役割:** 4bitの選択信号に基づき、16個の1bit入力から1つを出力します。

#### 【L1: 機能ブロック層】
L2のパーツを規則的に並べ、データの「まとまり」を制御します。

3.  **reg_file_bank_16.v**
    *   **役割:** `reg_file_cell_16bit` を16個並べ、書き込みデータ（WD）を全レジスタへ配線します。
    *   **構造:** `generate` 文を使用。ただし、**Index 0（ゼロレジスタ）については、書き込みを無効化し、出力を常に `0` に固定**する構造を明示的に記述します。
    *   **視覚化の利点:** 16個のレジスタが整列している様子が見え、かつ「レジスタ0だけ配線が特殊であること（ゼロレジスタ）」が回路図から読み取れます。
4.  **reg_file_read_port_16bit.v**
    *   **役割:** 16個のレジスタ値から、指定されたアドレスの値を出力します。
    *   **構造:** `generate` 文を用いて、`reg_file_mux_16to1_1bit` を16個並列化（16bit幅に拡張）します。
    *   **視覚化の利点:** 16個のレジスタから巨大なマルチプレクサへ線が集まっていく様子が視覚化されます。

#### 【L0: ユニット層（最上位）】
外部（CPUデータパス）とのインターフェースです。

5.  **reg_file.v**
    *   **役割:** 2つの読出ポート（RS1用、RS2用）と、1つの書込デコーダを結合します。
    *   **構成:** `reg_file_bank_16` を1つ、`reg_file_read_port_16bit` を2つ（2ポート同時読出のため）インスタンス化します。

---

### 3. 提案するサブモジュールリスト

submodules:
reg_file: レジスタファイルの最上位。バンク、読出ポートx2、書き込み制御を統合する。
reg_file_bank_16: 16個の16bitレジスタを保持する。レジスタ0は0固定、1-15は書き込み可能な記憶域として構成する。
reg_file_read_port_16bit: 16入力16bit出力のセレクタ。16個のレジスタ値から1つのアドレスに対応する値を選択する。
reg_file_cell_16bit: 16bit幅の同期書き込みレジスタ。レジスタファイル内の個別レジスタ1個分に相当する。
reg_file_mux_16to1_1bit: 16個の1bit入力から1つを選択する最小単位のマルチプレクサ。
reg_file_decoder_4to16: 4bitの書き込みアドレスをデコードし、バンク内の特定のレジスタに対する書き込み有効信号（16本）を生成する。
