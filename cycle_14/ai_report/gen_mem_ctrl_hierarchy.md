教育用シングルサイクルCPUにおいて、**mem_ctrl**（メモリコントローラ）は、CPU内部のデータパスと外部のデータメモリ（dmem）を仲介する役割を担います。

ISAで定義された `load` と `store` 命令における「メモリアドレス計算（rs1 + imm）」と「データの入出力方向」を視覚化するため、以下の3階層による構造化を提案します。

---

### 1. 階層構造の概要

| 階層レベル | モジュール名 | 役割 | パタン構造化の適用 |
| :--- | :--- | :--- | :--- |
| **L0 (ユニット)** | `mem_ctrl` | 最上位。アドレス計算器とデータバス・バッファを統合。 | 構成要素の接続 |
| **L1 (機能ブロック)** | `mem_ctrl_adder_16bit` | 16bit加算器。メモリ実行アドレス（rs1 + imm）を計算。 | `generate`によるフルアダーの連結 |
| | `mem_ctrl_buffer_16bit` | 16bitデータバッファ。メモリへ送るデータの流れを明示。 | `generate`による1bitバッファの並列化 |
| **L2 (プリミティブ)** | `mem_ctrl_full_adder` | 1ビット全加算器。 | 基本論理ゲート |
| | `mem_ctrl_buffer_1bit` | 1ビットバッファ（信号通過）。 | 基本論理ゲート |

---

### 2. 各ファイルの役割と階層レベルの説明

#### 【L2: プリミティブ層】
最小単位の回路素子です。

1.  **mem_ctrl_full_adder.v**
    *   **役割:** 1ビットの加算を行います。
2.  **mem_ctrl_buffer_1bit.v**
    *   **役割:** 1ビットの信号をそのまま通過させます（論理的には線と同じですが、回路図上で「メモリへの出口」をブロックとして表現するために使用します）。

#### 【L1: 機能ブロック層】
プリミティブを16ビット幅に拡張し、メモリアクセスの構造を具体化します。

3.  **mem_ctrl_adder_16bit.v**
    *   **役割:** `load`/`store`命令で使用する実効アドレス（Effective Address）を計算します。
    *   **構造:** `generate` 文を用いて `mem_ctrl_full_adder` を16個インスタンス化し、リップルキャリー構造を作ります。
    *   **視覚化の利点:** 回路図上で「レジスタの値と即値が加算されてアドレスになる」という流れが、独立した加算器ブロックとして表示されます。
4.  **mem_ctrl_buffer_16bit.v**
    *   **役割:** CPU内部のレジスタ値をメモリのデータ入力ポートへ導きます。
    *   **構造:** `generate` 文を用いて `mem_ctrl_buffer_1bit` を16個並列配置します。
    *   **視覚化の利点:** 回路図上で、16ビットのデータバスが整然とメモリに向かって流れる様子が強調されます。

#### 【L0: ユニット層（最上位）】
CPUコアと外部メモリ `dmem` を繋ぐインターフェースをまとめます。

5.  **mem_ctrl.v**
    *   **役割:** 以下の制御を統合します。
        1.  **アドレス生成:** `rs1` と `imm` を `adder` に入力し、結果を `o_addr_to_dmem` へ出力。
        2.  **書き込みデータ準備:** `rs2` の値をバッファを通して `o_data_to_dmem` へ出力。
        3.  **書き込み有効信号:** 制御ユニットからの信号を `o_dmem_wen` へバイパス。
        4.  **読み出しデータ:** メモリからの `i_data_from_dmem` をCPU内部へバイパス。

---

### 3. 提案するサブモジュールリスト

submodules:
mem_ctrl: メモリ制御ユニットの最上位。アドレス計算とデータ入出力を統合する。
mem_ctrl_adder_16bit: メモリの実効アドレス（rs1 + imm）を計算するための16ビット加算器。
mem_ctrl_buffer_16bit: メモリへ書き込む16ビットデータを整理して出力するためのバッファブロック。
mem_ctrl_full_adder: 加算器を構成する最小単位の1ビット全加算器。
mem_ctrl_buffer_1bit: データの流れを視覚化するための最小単位の1ビットバッファ。
