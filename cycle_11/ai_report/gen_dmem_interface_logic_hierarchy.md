「メモリへのアクセスアドレスの計算（rs1 + imm）」と「メモリへの書き込み制御信号」の生成プロセスを視覚化するため、`dmem_interface_logic`を計算ブロックと制御ブロックに階層化・パターン構造化することを提案します。

特に、メモリアドレス計算（rs1 + imm）を独立した加算器ブロックとして定義し、その内部をビットスライス化することで、RTL Viewer上で「ベースアドレスにオフセットが足し込まれる様子」を強調します。

### 1. 階層構造一覧（ファイル構成）

1ファイル1モジュールの原則に従い、以下の4つのファイルで構成します。

| レベル | ファイル名 | 役割 |
| :--- | :--- | :--- |
| **L0 (Top)** | `dmem_interface_logic.v` | 最上位モジュール。アドレス計算器と制御ロジックを統合し、メモリ(dmem)へのインターフェースを提供する。 |
| **L1 (Block)** | `dmem_interface_logic_adder_16bit.v` | メモリアドレス計算用の16bit加算器。`generate`文でフルアダーを16個配置する。 |
| **L2 (Cell)** | `dmem_interface_logic_full_adder.v` | 1bitの全加算器。アドレス計算の最小単位。 |
| **L3 (Atom)** | `dmem_interface_logic_half_adder.v` | 1bitの半加算器。加算器の最小論理構成要素。 |

---

### 2. 各モジュールの役割と階層レベルの詳細

#### L0: `dmem_interface_logic.v`
CPUのパイプラインとデータメモリ(dmem)の橋渡しをします。
- **アドレス計算の統合**: レジスタ(rs1)と符号拡張された即値(imm)を加算器へ渡し、メモリ用アドレス信号（`o_addr_to_dmem`）を生成します。
- **書き込み有効制御**: 命令デコーダからの`o_mem_write_en`をそのままメモリへ通し、Store命令時のみ書き込みが発生するよう制御します。
- **データパスの接続**: レジスタ(rs2)の値を、メモリの書き込みデータピン（`o_data_to_dmem`）へバイパスします。

#### L1: `dmem_interface_logic_adder_16bit.v` (パターン構造化)
ISA定義の `Mem[rs1+imm]` という記述のうち、`rs1+imm` の部分を物理的な回路として視覚化します。
- **構造**: `generate`文を用いて、`dmem_interface_logic_full_adder` を16個生成し、キャリーを連鎖させます。
- **視覚的効果**: 合成後の回路図で、16個の同じ「箱」が並び、メモリの最終的なアドレスが算出されるまでの信号の伝播が一本のパスとして表現されます。

#### L2/L3: `full_adder` と `half_adder`
アドレス計算の精度と論理構造を教育的に示します。
- ALUの加算器と同じ構造を「メモリインターフェース専用」に持つことで、Load/Storeが専用の計算リソースを必要としているアーキテクチャの贅沢さ（シングルサイクルの特徴）を視覚的に提示できます。

---

### 3. モジュール呼び出しのイメージ（パタン構造化）

`dmem_interface_logic_adder_16bit.v` 内での実装イメージです。

```verilog
wire [16:0] w_carry;
assign w_carry[0] = 1'b0; // オフセット加算のベースキャリーは0

genvar i;
generate
    for (i = 0; i < 16; i = i + 1) begin : gen_dmem_addr_adder
        dmem_interface_logic_full_adder u_fa (
            .i_a    (i_rs1_data[i]),
            .i_b    (i_imm_data[i]),
            .i_cin  (w_carry[i]),
            .o_sum  (o_calculated_addr[i]),
            .o_cout (w_carry[i+1])
        );
    end
endgenerate
```

### 4. この階層化による教育的利点

1.  **「アドレス計算」の物理的実体の理解**: メモリの番地が、プログラム上の数値ではなく「電気的な加算結果」として確定していく様子を視覚化できます。
2.  **Load/Store命令の対称性**: メモリへのアドレス供給は共通で、書き込み許可信号（WE）だけが異なるという構造が、回路図上で一目でわかります。
3.  **データメモリへの入出力分離**: 「計算したアドレス」「レジスタから持ってきたデータ」「デコーダが作った制御信号」という3系統の信号が、メモリ(dmem)という1つのコンポーネントに入っていく様子が物理的にマッピングされます。
