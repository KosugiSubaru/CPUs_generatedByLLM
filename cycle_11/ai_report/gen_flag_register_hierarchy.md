「1クロック前の演算結果を参照する」というISAの仕様を視覚化するために、`flag_register`をフラグごとに独立したビットセルとして階層化・パターン構造化することを提案します。

これにより、論理合成後の回路図で、ALUから出力された複数のフラグがそれぞれ独立したレジスタ（D-FF）に吸い込まれ、1クロック遅れて出力される様子が明確に表現されます。

### 1. 階層構造一覧（ファイル構成）

1ファイル1モジュールの原則に従い、以下の3つのファイルで構成します。

| レベル | ファイル名 | 役割 |
| :--- | :--- | :--- |
| **L0 (Top)** | `flag_register.v` | 最上位モジュール。Z, N, Vの3つのフラグ入力をビットセルへ分配・集約する。 |
| **L1 (Cell)** | `flag_register_bit.v` | 1bitのフラグ保持ユニット（書き込み有効信号付きD-FF）。 |
| **L1 (Cell)** | `flag_register_mux_2to1_1bit.v` | （必要に応じて）書き込み有効信号を実現するための1bitセレクタ。 |

---

### 2. 各モジュールの役割と階層レベルの詳細

#### L0: `flag_register.v`
全体の統合を行います。
- ALUから計算されたフラグ信号（Z, N, V）をまとめて受け取ります。
- `generate`文を使用して、フラグの数（3個）だけ`flag_register_bit`をインスタンス化します。
- デコーダからの「フラグ更新有効信号（`i_wen`）」を全てのビットセルに供給します。
- **視覚的効果**: 3つの独立した箱（フラグ）が並列に並び、一斉に更新される様子が回路図上で確認できます。

#### L1: `flag_register_bit.v`
フラグの最小単位となる記憶素子です。
- 内部にクロック同期のレジスタを持ちます。
- 書き込み有効信号（`i_we`）が `1` のときのみ、入力を取り込みます。
- **教育的意図**: このモジュールが介在することで、分岐命令が「現在計算中のフラグ」ではなく「1クロック前にレジスタに閉じ込められたフラグ」を参照していることを視覚的に強調します。

#### L1: `flag_register_mux_2to1_1bit.v`
レジスタの更新制御（書き込み有効信号）を論理ゲートレベルで見せるために使用します。
- `i_we` が `0` のときは現在の値を保持（フィードバック）、`1` のときは新しい値を選択します。
- これを `flag_register_bit.v` 内で呼び出します。

---

### 3. モジュール呼び出しのイメージ（パタン構造化）

`flag_register.v` (L0) 内での実装イメージです。

```verilog
// Z, N, Vの3bit分をループで生成
genvar i;
generate
    for (i = 0; i < 3; i = i + 1) begin : gen_flag_bits
        flag_register_bit u_bit (
            .i_clk   (i_clk),
            .i_rst_n (i_rst_n),
            .i_we    (i_wen),      // 命令デコーダからのフラグ更新許可信号
            .i_d     (i_flags[i]), // ALUからのフラグ入力
            .o_q     (o_flags[i])  // 次のクロックの分岐判定へ
        );
    end
endgenerate
```

### 4. この階層化による教育的利点

1.  **「ステータスレジスタ」の存在感**: 1つの16bitレジスタの一部としてではなく、Z, N, Vという特定の役割を持った独立したビットの集まりとして視認できます。
2.  **更新タイミングの理解**: 全ての命令がフラグを更新するわけではない（例：分岐命令自体はフラグを更新しない）ため、`i_wen` 信号がどの命令のときに `1` になるかを回路図の配線を追うことで理解しやすくなります。
3.  **1クロック遅延の視覚化**: ALUと分岐判定ロジック（次アドレス生成）の間に必ずこのビットセルの「箱」が挟まるため、パイプラインの原型のような「データの蓄積と遅延」を直感的に捉えられます。
