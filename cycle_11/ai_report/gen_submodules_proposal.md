ご提示いただいたISA定義とシステムアーキテクチャに基づき、教育用シングルサイクルCPUを構成するサブモジュールとその動作について提案します。

この構成は、命令の実行フロー（Fetch, Decode, Execute, Memory, Write Back）を視覚的に追いやすいよう分割しています。

---

### 1. CPUを構成するサブモジュール一覧

| モジュール名 | 役割・機能説明 |
| :--- | :--- |
| **PC (Program Counter)** | 次に実行する命令のアドレスを保持するレジスタ。 |
| **Control Unit (デコーダ)** | 16bit命令の`opcode`を解析し、ALU演算の選択、レジスタ書き込み有効化、メモリ操作などの制御信号を生成する。 |
| **Register File** | 16個の16bitレジスタを保持する。2つの読み出しポートと1つの書き込みポートを持つ。レジスタ0番は常に0を出力する。 |
| **ALU (Arithmetic Logic Unit)** | 加減算、論理演算、シフト演算を行う。演算結果とともに、フラグ（Z, N, V）を出力する。 |
| **Flag Register** | ALUが出力したフラグを保存する。ISAの規定通り、分岐命令は「1クロック前」のフラグを参照するため、クロック同期で更新する。 |
| **Immediate Generator** | 命令内のビットフィールドから即値（4bit/8bit/12bit）を取り出し、16bitに符号拡張する。 |
| **Branch/Jump Unit** | `blt`や`bz`の条件判定を行い、次のPCを「PC+2」にするか「PC+imm（分岐）」にするかを選択する。 |
| **Data Selector (Mux)** | 各ステージで、ALUへの入力を「レジスタ」にするか「即値」にするか、レジスタへの書き戻し値を「ALU結果」「メモリ出力」「PC+2」のどれにするかを選択する複数のマルチプレクサ。 |

---

### 2. CPUの動作フローと相互作用

シングルサイクルCPUとして、1クロックの間に以下の処理が連鎖的に行われます。

#### ① Fetch (命令取得)
- **PC**が指すアドレスを命令メモリ（imem）へ出力し、命令（`w_instr`）を取得します。
- 同時に、**PC+2**（次の命令アドレス）を計算しておきます。

#### ② Decode (解読とレジスタ読み出し)
- **Control Unit**が命令の下位4bit（opcode）を見て、どの演算を行うか、メモリを使用するか等を判断します。
- **Register File**から、命令で指定された`rs1`, `rs2`のアドレスに対応するデータを取り出します。
- **Immediate Generator**が、命令形式に合わせて即値を16bitに符号拡張します。

#### ③ Execute (演算とフラグ更新)
- **ALU**が、レジスタから読み出した値（または即値）を用いて演算（ADD, SUB, AND等）を実行します。
- 演算結果からZ（ゼロ）、N（負）、V（オーバーフロー）のフラグが生成されます。
- **Flag Register**は、このフラグを次のクロックの立ち上がりで保存します（現在の分岐命令は、前回の命令が保存したフラグを参照します）。

#### ④ Memory / Write Back (メモリ操作と書き戻し)
- **Load/Store命令の場合**: ALUで計算されたアドレスを使用して、データメモリ（dmem）から値を読み出す、もしくはレジスタの値を書き込みます。
- **書き戻し**:
    - 通常演算なら「ALUの結果」
    - Load命令なら「メモリから読み出した値」
    - JAL/JALR命令なら「PC+2（戻り先アドレス）」
    - これらの中から適切なものを選択し、**Register File**の`rd`へ書き込みます。

#### ⑤ Next PC (次命令アドレスの決定)
- **Branch/Jump Unit**が、現在の命令が分岐・ジャンプ命令かどうか、およびフラグの状態（Z, N^Vなど）を確認します。
- 条件が成立していれば `PC + imm`（または `rs1 + imm`）、成立していなければ `PC + 2` を次のPC値として決定し、次のクロックで**PC**を更新します。

### 教育的な視点でのポイント
- **フラグの遅延:** 「1クロック前のフラグを参照する」という仕様は、回路図上ではALUと分岐判定ユニットの間に「Flag Register」という箱（Dフリップフロップ）が介在することを意味し、タイミング設計の学習に最適です。
- **データパスの可視化:** ALUの出力がそのままメモリのアドレスになり、メモリの出力がレジスタに戻っていく様子をモジュール分割することで、データがCPU内をどう流れるかを視覚的に理解しやすくなります。
