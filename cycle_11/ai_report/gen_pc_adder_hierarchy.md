「次アドレスの計算」や「分岐ターゲットの計算」に使用される`pc_adder`モジュールを、加算の仕組みを論理ゲートレベルから視覚的に理解できるように階層化・パターン構造化して提案します。

Ripple Carry Adder（リップルキャリーアダー）方式を採用し、下位ビットから上位ビットへキャリー（繰り上がり）が伝播していく様子を回路図上で強調します。

### 1. 階層構造一覧（ファイル構成）

1ファイル1モジュールの原則に従い、以下の4つのファイルで構成します。

| レベル | ファイル名 | 役割 |
| :--- | :--- | :--- |
| **L0 (Top)** | `pc_adder.v` | 加算器のトップ。16bitの入力A, Bを受け取り、和（Sum）を出力する。 |
| **L1 (Block)** | `pc_adder_16bit.v` | `generate`文を用いて、フルアダーを16個数珠つなぎに配置する。 |
| **L2 (Cell)** | `pc_adder_full_adder.v` | 1bitの全加算器。2つの半加算器とORゲートで構成される。 |
| **L3 (Atom)** | `pc_adder_half_adder.v` | 1bitの半加算器。加算の最小単位（XORとAND）。 |

---

### 2. 各モジュールの役割と階層レベルの詳細

#### L0: `pc_adder.v`
CPUのデータパスから呼び出されるインターフェースです。
- 内部的には `pc_adder_16bit` をインスタンス化します。
- キャリー入力（cin）を `0` に固定することで、純粋な加算器として機能させます。

#### L1: `pc_adder_16bit.v` (パターン構造化の核心)
16ビットの加算ロジックをビットごとに展開します。
- **構造**: `genvar`と`generate`文を使用し、`pc_adder_full_adder`を16個生成します。
- **配線**: ビット `i` のキャリー出力をビット `i+1` のキャリー入力に接続します。
- **視覚的効果**: 合成後の回路図（RTL Viewer）で、16個の同じブロックが横に並び、キャリー信号が右から左（下位から上位）へ一本の線で繋がっている様子が明確に見えます。

#### L2: `pc_adder_full_adder.v`
1ビットの加算と、前段からのキャリー処理を行います。
- 内部で2つの `pc_adder_half_adder` をインスタンス化します。
- **教育的意図**: 「全加算器は半加算器2つでできている」という教科書的な構成をそのまま回路ブロックとして表現します。

#### L3: `pc_adder_half_adder.v`
加算の最小構成単位です。
- XORゲート（和）とANDゲート（キャリー）を記述します。
- **視覚的効果**: 加算器の最深部で、どのような論理ゲートが働いているかを確認できます。

---

### 3. モジュール呼び出しのイメージ（パタン構造化）

`pc_adder_16bit.v` 内での実装イメージです。

```verilog
wire [16:0] w_carry;
assign w_carry[0] = 1'b0; // 初期キャリーは0

genvar i;
generate
    for (i = 0; i < 16; i = i + 1) begin : gen_ripple_carry
        pc_adder_full_adder u_fa (
            .i_a    (i_a[i]),
            .i_b    (i_b[i]),
            .i_cin  (w_carry[i]),
            .o_sum  (o_sum[i]),
            .o_cout (w_carry[i+1])
        );
    end
endgenerate
```

### 4. この階層化による教育的利点

1.  **キャリー伝播の可視化**: Ripple Carry Adderの構造をビットごとに分割してインスタンス化することで、加算器の内部で「計算の待ち時間（遅延）」が発生する物理的な理由（キャリーの伝達）を、回路図の配線構造として理解できます。
2.  **数学的構造の理解**: 加算という操作が、単純な1bitの論理（半加算器）の積み重ねで16bitまで拡張されている様子を、階層を辿ることで追体験できます。
3.  **再利用性の提示**: `pc_adder` が `PC + 2`（定数加算）にも `PC + imm`（変数加算）にも、全く同じ「加算器」というブロックで対応できる汎用性を視覚的に示せます。
