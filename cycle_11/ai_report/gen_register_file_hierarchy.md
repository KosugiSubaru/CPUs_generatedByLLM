「レジスタファイルの構造を視覚的に理解する」という目的に基づき、`register_file`を階層化・パターン構造化して提案します。

レジスタファイルは「16個の貯蔵庫（レジスタ）」と「そこから目的の値を取り出す巨大な選択器（マルチプレクサ）」で構成されます。これをビット単位まで分解することで、回路図上でデータがどのように格納され、どのように抽出されるかを視覚的に表現します。

### 1. 階層構造一覧（ファイル構成）

1ファイル1モジュールの原則に従い、以下の5つのファイルで構成します。

| レベル | ファイル名 | 役割 |
| :--- | :--- | :--- |
| **L0 (Top)** | `register_file.v` | 全体の統合。16個のレジスタスロットと、2つの読出し用セレクタを接続。 |
| **L1 (Block)** | `register_file_slot.v` | 16bit幅の書き込み機能付きレジスタ。1bitセルを16個並列配置。 |
| **L1 (Block)** | `register_file_read_selector.v` | 16個のレジスタから1つを選ぶ16入力16bitマルチプレクサ。 |
| **L2 (Cell)** | `register_file_bit_cell.v` | 書き込み有効（WE）付き1bit D-FF。レジスタの最小記憶単位。 |
| **L2 (Cell)** | `register_file_mux_16to1_1bit.v` | 16個の入力信号から1つを抽出する1bitマルチプレクサ。 |

---

### 2. 各モジュールの役割と階層レベルの詳細

#### L0: `register_file.v`
最上位モジュールです。
- `generate`文を用いて、`register_file_slot`を16個インスタンス化します。
- **ゼロレジスタの処理**: ISA定義に従い、0番目のスロットへの書き込みは無効化し、読み出し時は常に0を出力するよう配線します。
- `read_selector`を2つ配置し、ポート1（rs1）とポート2（rs2）を独立して出力します。

#### L1/L2: `register_file_slot.v` と `bit_cell.v` (書き込み構造)
データの「保存」を視覚化します。
- **`bit_cell`**: クロックに同期し、`we`（書き込み有効）が1の時だけデータを取り込む1bitの箱です。
- **`register_file_slot`**: `generate`文で1bitセルを16個横に並べます。
- **視覚的効果**: 回路図では、16bitのデータ線が各ビットのセルに整然と分岐していく様子が確認できます。

#### L1/L2: `register_file_read_selector.v` と `mux_16to1_1bit.v` (読み出し構造)
データの「抽出」を視覚化します。
- **`mux_16to1_1bit`**: 16個のレジスタの「同じビット位置」の信号（例：全レジスタのbit0）を集め、アドレスに応じて1つを選びます。
- **`register_file_read_selector`**: `generate`文で1bitマルチプレクサを16個並列に並べます。
- **視覚的効果**: 16個のレジスタの出力が、巨大なバスのようにセレクタに集まり、そこから16bitが絞り出される様子が回路図上で強調されます。

---

### 3. モジュール呼び出しのイメージ（パタン構造化）

`register_file.v`（L0）において、16個のレジスタを並べる際のイメージです。

```verilog
genvar i;
generate
    for (i = 0; i < 16; i = i + 1) begin : gen_reg_slots
        // 0番レジスタは定数0、それ以外は通常のスロット
        if (i == 0) begin : zero_reg
            assign w_reg_outputs[0] = 16'h0000;
        end else begin : normal_reg
            register_file_slot u_slot (
                .i_clk   (i_clk),
                .i_rst_n (i_rst_n),
                .i_we    (i_rd_addr == i && i_wen), // 自分宛の書き込みか判定
                .i_data  (i_rd_data),
                .o_data  (w_reg_outputs[i])
            );
        end
    end
endgenerate
```

### 4. この階層化による教育的利点

1.  **「レジスタ0 = 0」の物理的理解**: 回路図上で、0番目のスロットだけが他と異なり、出力がグランド（0）に固定されている、あるいは書き込み線が繋がっていない様子を視覚的に示せます。
2.  **マルチポート読み出しの理解**: なぜ同時に2つのレジスタを読み出せるのか？ それは同じ16個の箱に対して、2セットの巨大なセレクタ（MUX）が独立して繋がっているからだ、という物理構造が明確になります。
3.  **アドレスデコードの可視化**: 4bitのアドレスがどのように各スロットの`we`信号やMUXの選択信号に影響を与えるかを、配線の広がりとして捉えることができます。
