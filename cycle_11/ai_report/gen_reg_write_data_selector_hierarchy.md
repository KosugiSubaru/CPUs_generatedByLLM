「レジスタへの書き戻しデータ（ALU演算結果、メモリ読出しデータ、PC+2、即値）をどのように選択するか」を視覚化するため、`reg_write_data_selector`をビットスライス構造に階層化・パターン構造化して提案します。

この構成により、論理合成後の回路図で、複数の異なるデータパス（演算器、メモリ、プログラムカウンタ）が1つの地点（レジスタ入力）へ集約され、制御信号によって1つに絞り込まれる様子が明確に表現されます。

### 1. 階層構造一覧（ファイル構成）

1ファイル1モジュールの原則に従い、以下の3つのファイルで構成します。

| レベル | ファイル名 | 役割 |
| :--- | :--- | :--- |
| **L0 (Top)** | `reg_write_data_selector.v` | 最上位モジュール。各データソース（ALU, Mem, PC, Imm）を統合する。 |
| **L1 (Block)** | `reg_write_data_selector_mux_4to1_16bit.v` | 16bit幅の4入力マルチプレクサ。`generate`文で1bitセルを16個並列配置。 |
| **L2 (Cell)** | `reg_write_data_selector_mux_4to1_1bit.v` | 書き戻しデータ選択の最小単位となる1bit 4入力マルチプレクサ。 |

---

### 2. 各モジュールの役割と階層レベルの詳細

#### L0: `reg_write_data_selector.v`
レジスタファイルの`rd`入力に接続される直前の「最終セレクタ」です。
- 命令デコーダからの制御信号（例：`i_wb_sel[1:0]`）を受け取ります。
- **データソースの統合**: 
    1. ALUの演算結果 (`add`, `sub` 等)
    2. データメモリからの読み出し値 (`load`)
    3. 次の命令アドレス (`jal`, `jalr` のリンク用 PC+2)
    4. 拡張された即値 (`loadi`)
- これらをL1ブロックへ渡し、最終的な16bitデータを出力します。

#### L1: `reg_write_data_selector_mux_4to1_16bit.v` (パターン構造化)
16ビットのバス選択を、独立した1ビットのスイッチ16個の集合として定義します。
- **構造**: `generate`文を用いて、`reg_write_data_selector_mux_4to1_1bit` を16個生成します。
- **視覚的効果**: 16個の同じ「箱」が並び、共通の「選択信号」によって全てのビットが一斉に切り替わる様子が見えます。これは「データ幅が16bitである」というアーキテクチャの物理的実態を視覚的に強調します。

#### L2: `reg_write_data_selector_mux_4to1_1bit.v`
選択ロジックの最小単位です。
- 4つの入力（d0〜d3）から、2bitの選択信号（sel）に応じて1つを出力します。
- **教育的意図**: 回路の最下層において、情報の「合流と選択」がどのように行われているかを最小単位で示します。

---

### 3. モジュール呼び出しのイメージ（パタン構造化）

`reg_write_data_selector_mux_4to1_16bit.v` 内での実装イメージです。

```verilog
genvar i;
generate
    for (i = 0; i < 16; i = i + 1) begin : gen_wb_mux_bits
        reg_write_data_selector_mux_4to1_1bit u_mux_bit (
            .i_sel (i_sel),
            .i_d0  (i_alu_res[i]),
            .i_d1  (i_mem_data[i]),
            .i_d2  (i_pc_plus_2[i]),
            .i_d3  (i_imm_data[i]),
            .o_y   (o_wb_data[i])
        );
    end
endgenerate
```

### 4. この階層化による教育的利点

1.  **書き戻し（Write Back）概念の可視化**: 命令によって「レジスタに書き込まれる値の正体」が異なることを、物理的なMUXの入力系統として理解できます。
2.  **データパスの交差点の理解**: ALU、メモリ、PCという異なるユニットの出力が、このセレクタを介してレジスタファイルへと戻っていく「環状構造」を回路図上で追跡しやすくなります。
3.  **ビット並列制御の提示**: 1bitのMUXを16個並べることで、16bitアーキテクチャがいかに規則正しいセルの反復で構成されているかを視覚的に提示できます。
