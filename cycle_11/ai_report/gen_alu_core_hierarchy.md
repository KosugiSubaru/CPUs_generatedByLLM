「演算器（ALU）がどのようにデータを選択し、処理しているか」を視覚化するために、`alu_core`モジュールを機能単位（算術、論理、シフト）およびビットスライス構造に階層化・パターン構造化することを提案します。

### 1. 階層構造一覧（ファイル構成）

1ファイル1モジュールの原則に従い、以下の8つのファイルで構成します。

| レベル | ファイル名 | 役割 |
| :--- | :--- | :--- |
| **L0 (Top)** | `alu_core.v` | 算術・論理・シフトの各ブロックを統合し、フラグ（Z, N, V）を生成する。 |
| **L1 (Block)** | `alu_core_arithmetic_16bit.v` | 加減算を担当。フルアダーの集合体として構成。 |
| **L1 (Block)** | `alu_core_logic_16bit.v` | AND/OR/XOR/NOTの論理演算をビット並列で実行。 |
| **L1 (Block)** | `alu_core_shifter_16bit.v` | SRA/SLA（算術右シフト/左シフト）を実行。 |
| **L1 (Block)** | `alu_core_mux_8to1_16bit.v` | opcodeに基づき、最終的な演算結果を16bit幅で選択。 |
| **L2 (Cell)** | `alu_core_full_adder.v` | 算術演算の最小単位。加算器のキャリー連鎖を視覚化。 |
| **L2 (Cell)** | `alu_core_logic_slice.v` | 論理演算の最小単位。1bitごとのビット独立演算を視覚化。 |
| **L2 (Cell)** | `alu_core_mux_8to1_1bit.v` | 結果選択の最小単位。1bitごとの選択ロジック。 |

---

### 2. 各モジュールの役割と階層レベルの詳細

#### L0: `alu_core.v`
演算の全体像を示します。各L1ブロックにデータ（rs1, rs2）を配り、その結果を集めます。
- **フラグ生成**: 演算結果のbit[15]からNフラグを、結果が0かどうかからZフラグを、算術ブロックからの信号でVフラグをそれぞれ生成します。

#### L1/L2: `alu_core_arithmetic_16bit.v` と `full_adder.v` (算術回路)
- **構造**: `generate`文を用いて`full_adder`を16個並列配置します。
- **減算の視覚化**: 減算(`0001`)のとき、`rs2`を反転させ、最下位ビットの`cin`を1にすることで「2の補数」による加算で減算を実現している様子を配線で示します。

#### L1/L2: `alu_core_logic_16bit.v` と `logic_slice.v` (論理回路)
- **構造**: ビットごとに独立した論理演算を行う`logic_slice`を16個配置します。
- **視覚的効果**: 論理演算がビット間で干渉せず、並列に処理されている様子が回路図上で明確になります。

#### L1: `alu_core_shifter_16bit.v` (シフト回路)
- **構造**: `rs1`の各ビットが`rs2`（シフト量）に応じてどのように移動するかを記述します。
- **教育的意図**: 算術シフト（符号ビットの維持）がどのように配線されているかを強調します。

#### L1/L2: `alu_core_mux_8to1_16bit.v` と `mux_8to1_1bit.v` (結果選択器)
- **構造**: 16bit分の「演算結果の出口」をビットごとにマルチプレクサで構成します。
- **教育的意図**: 常に全ての演算（加算も論理演算も）が行われており、最後のMUXで1つだけ選ばれているという「ハードウェア的な並列処理」の概念を視覚化します。

### 3. モジュール呼び出しのイメージ（パタン構造化）

`alu_core_arithmetic_16bit.v` でのフルアダー接続のイメージです。

```verilog
wire [16:0] w_carry;
assign w_carry[0] = i_is_sub; // 減算時は+1（2の補数）

genvar i;
generate
    for (i = 0; i < 16; i = i + 1) begin : gen_arithmetic_slice
        alu_core_full_adder u_fa (
            .a    (i_rs1[i]),
            .b    (i_rs2[i] ^ i_is_sub), // 減算時は反転
            .cin  (w_carry[i]),
            .sum  (o_res[i]),
            .cout (w_carry[i+1])
        );
    end
endgenerate
```

### 4. この階層化による教育的利点

1.  **「符号付き演算」の物理的実装**: V（オーバーフロー）フラグが、最上位ビット（MSB）のキャリー入力とキャリー出力の異同によって決まる様子などを、加算器の末端の配線として視覚的に示せます。
2.  **演算の独立性**: CPUが「まず加算してから、次に論理演算をする」のではなく、「全ての演算器にデータが流れ、最後にMUXで選んでいる」という同時性を理解させやすくなります。
3.  **ビットスライスの規則性**: 大規模な回路も、単純な1bitの部品を並べることで構成されているという、デジタル回路設計の王道を提示できます。
