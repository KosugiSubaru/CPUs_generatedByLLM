「命令形式ごとの即値の取り出しと符号拡張」を視覚化するため、`imm_generator`モジュールを形式別の拡張ユニットと、それらを選択するマルチプレクサに階層化・パターン構造化して提案します。

この構成により、論理合成後の回路図で「どのビットが符号ビットとしてコピーされているか」「命令ごとにどの範囲のビットが即値として扱われているか」がブロックとして明確に分離されます。

### 1. 階層構造一覧（ファイル構成）

1ファイル1モジュールの原則に従い、以下の7つのファイルで構成します。

| レベル | ファイル名 | 役割 |
| :--- | :--- | :--- |
| **L0 (Top)** | `imm_generator.v` | 全体の統合。命令形式に応じた各拡張ユニットの呼び出しと最終選択。 |
| **L1 (Block)** | `imm_generator_ext4a_16bit.v` | 命令[7:4]を即値（4bit）として取り出し、16bitに符号拡張する（load/addi用）。 |
| **L1 (Block)** | `imm_generator_ext4b_16bit.v` | 命令[15:12]を即値（4bit）として取り出し、16bitに符号拡張する（store用）。 |
| **L1 (Block)** | `imm_generator_ext8_16bit.v` | 命令[11:4]を即値（8bit）として取り出し、16bitに符号拡張する（loadi/jal用）。 |
| **L1 (Block)** | `imm_generator_ext12_16bit.v` | 命令[15:4]を即値（12bit）として取り出し、16bitに符号拡張する（branch用）。 |
| **L1 (Block)** | `imm_generator_mux_4to1_16bit.v` | 命令形式（セレクタ信号）に基づき、上記4つの拡張結果から1つを選択する。 |
| **L2 (Cell)** | `imm_generator_mux_4to1_1bit.v` | 即値選択の最小単位となる1bitマルチプレクサ。 |

---

### 2. 各モジュールの役割と階層レベルの詳細

#### L0: `imm_generator.v`
命令デコーダから渡される「即値形式選択信号」に基づき、適切な即値を生成します。
- 全ての拡張ユニットを並列に動かし、最後にマルチプレクサで1つを選びます。

#### L1: `imm_generator_extXX_16bit.v` (符号拡張ユニット)
ISAで定義された各命令形式に対応します。
- **4bit A型 ([7:4])**: `addi`, `load`, `jalr` で使用。MSBであるbit[7]を上位ビットへ複製。
- **4bit B型 ([15:12])**: `store` で使用。MSBであるbit[15]を上位ビットへ複製。
- **8bit 型 ([11:4])**: `loadi`, `jal` で使用。MSBであるbit[11]を上位ビットへ複製。
- **12bit 型 ([15:4])**: `blt`, `bz` で使用。MSBであるbit[15]を上位ビットへ複製。
- **視覚的効果**: 回路図上で「どのビットが符号拡張の根拠（MSB）になっているか」が配線の分岐として視覚化されます。

#### L1/L2: `imm_generator_mux_4to1_16bit.v` と `mux_4to1_1bit.v` (選択器)
- **構造**: `generate`文を用いて、1bitマルチプレクサを16個並列に配置します。
- **教育的意図**: 16bit全てのビットにおいて、同じ選択制御信号（命令形式）によって即値が選ばれているというビット並列性を視覚化します。

### 3. モジュール呼び出しのイメージ（パタン構造化）

`imm_generator_mux_4to1_16bit.v` 内での実装イメージです。

```verilog
genvar i;
generate
    for (i = 0; i < 16; i = i + 1) begin : gen_mux_bits
        imm_generator_mux_4to1_1bit u_mux (
            .i_sel (i_sel),     // どの命令形式か
            .i_d0  (i_imm4a[i]),
            .i_d1  (i_imm4b[i]),
            .i_d2  (i_imm8[i]),
            .i_d3  (i_imm12[i]),
            .o_y   (o_imm[i])
        );
    end
endgenerate
```

### 4. この階層化による教育的利点

1.  **「符号拡張」の物理的意味の理解**: 符号拡張とは、単に「0で埋める」のではなく、「特定の1本のワイヤ（MSB）を複数の入力ピンに接続する」ことであるという物理的実態を回路図で示せます。
2.  **命令フォーマットの多重性の可視化**: 命令語の同じビット位置が、ある命令ではレジスタ番号として、別の命令では即値の一部として扱われる「ビットの再利用」の様子が、各拡張ユニットへの配線の重なりとして理解できます。
3.  **ハードウェアの即時性**: セレクタで選ぶ前に全ての可能性（4/8/12bit拡張）が既に計算・準備されているという、シングルサイクルCPUの並列動作性を強調できます。
