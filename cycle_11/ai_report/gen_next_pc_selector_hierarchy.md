「次サイクルのプログラムカウンタ（PC）をどのように決定するか」を視覚化するため、`next_pc_selector` モジュールを、条件判定ロジックとデータ選択器（MUX）に階層化・パターン構造化して提案します。

この構成により、論理合成後の回路図で「フラグと命令が組み合わさって分岐の成否が決まり、それが巨大なスイッチ（MUX）を切り替える」という流れが直感的に理解できるようになります。

### 1. 階層構造一覧（ファイル構成）

1ファイル1モジュールの原則に従い、以下の4つのファイルで構成します。

| レベル | ファイル名 | 役割 |
| :--- | :--- | :--- |
| **L0 (Top)** | `next_pc_selector.v` | 全体の統合。分岐判定結果に基づいて、最終的な次PCアドレスを決定する。 |
| **L1 (Block)** | `next_pc_selector_br_eval.v` | **分岐判定ユニット**。命令（opcode）とフラグ（Z, N, V）を突き合わせ、分岐条件が成立しているかを判定する。 |
| **L1 (Block)** | `next_pc_selector_mux_16bit.v` | **16bit MUXブロック**。`generate`文を用いて、1bit MUXを16個並列配置する。 |
| **L2 (Cell)** | `next_pc_selector_mux_1bit.v` | **1bit MUXセル**。選択信号に基づき、複数の入力から1ビットを抽出する最小単位。 |

---

### 2. 各モジュールの役割と階層レベルの詳細

#### L0: `next_pc_selector.v`
このモジュールはPC更新の「意思決定」と「実行」を統括します。
- `br_eval` からの判定結果（分岐するか否か）と、デコーダからのジャンプ信号を組み合わせ、最終的な `pc_source`（MUXの切り替え信号）を生成します。
- その信号を `mux_16bit` に送り、次サイクルのアドレスを確定させます。

#### L1: `next_pc_selector_br_eval.v`（ISAロジックの視覚化）
ISAで定義された条件分岐の条件をそのまま回路にしたブロックです。
- **blt (1100)**: `N ^ V`（符号ビットの排他的論理和）が `1` なら分岐。
- **bz (1101)**: `Z`（ゼロフラグ）が `1` なら分岐。
- **視覚的効果**: 回路図上で、特定のopcodeがアクティブな時だけフラグがゲートを通り、`branch_taken` 信号が立ち上がる様子が見えます。

#### L1/L2: `next_pc_selector_mux_16bit.v` と `mux_1bit.v`（データパスの視覚化）
16bitの太いバスを切り替える巨大なスイッチです。
- **構造**: `generate`文を用いて、`next_pc_selector_mux_1bit` を16個配置します。
- **入力ソース**:
    1. `PC + 2`（通常実行）
    2. `PC + imm`（相対分岐・JAL）
    3. `rs1 + imm`（JALR）
- **視覚的効果**: 16ビット全てのビットが同じルールで一斉に切り替わる「ビット並列性」を強調します。

### 3. モジュール呼び出しのイメージ（パタン構造化）

`next_pc_selector_mux_16bit.v` 内での実装イメージです。

```verilog
genvar i;
generate
    for (i = 0; i < 16; i = i + 1) begin : gen_pc_mux_bits
        next_pc_selector_mux_1bit u_mux_bit (
            .i_sel (i_pc_source_sel), // どのソースを選ぶかの制御信号
            .i_d0  (i_pc_plus_2[i]),
            .i_d1  (i_pc_target[i]),  // PC + imm
            .i_d2  (i_rs1_target[i]), // rs1 + imm
            .i_d3  (16'h0000),        // (予約)
            .o_y   (o_next_pc[i])
        );
    end
endgenerate
```

### 4. この階層化による教育的利点

1.  **「条件分岐」のハードウェア的実体の理解**: 分岐とは「プログラムがどこかへ飛ぶ」ことではなく、回路的には「次のアドレスの供給源を、いつもの+2から、計算された即値側に切り替える（MUXを切り替える）」ことであるという物理的実態を理解できます。
2.  **フラグとPCの相関**: `blt`の条件である `N ^ V` がどのように回路で組まれているか、またそれがどうしてPCの選択に繋がっているのかという「制御信号の流れ」を回路図で追うことができます。
3.  **ジャンプ命令の統一的扱い**: JAL命令（無条件ジャンプ）と条件分岐命令（条件付きジャンプ）が、最終的には同じ「アドレス選択MUX」の異なる入力として統合されている美しさを視覚化できます。
