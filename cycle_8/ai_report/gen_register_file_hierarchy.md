「register_file（レジスタファイル）」モジュールを、論理合成後の回路図で「データの保存場所」と「読み出し・書き込みの選択機構」がISAの定義通りに視覚化されるよう階層化・パターン構造化します。

この設計では、**「16個の独立したレジスタ（箱）」**、**「書き込み先を決めるデコーダ」**、**「読み出しデータを選ぶ巨大なセレクタ」**を明確に分離します。

---

### 1. 階層構造の概要

| 階層レベル | ファイル名 | 役割 |
| :--- | :--- | :--- |
| **Level 0** | `register_file.v` | **最上位統合**: 書き込みデコーダ、レジスタバンク、読み出しセレクタを統合する。 |
| **Level 1** | `register_file_bank.v` | **レジスタバンク**: 16個の16bitレジスタを並列に配置。R0が常に0である論理もここで視覚化。 |
| **Level 1** | `register_file_read_mux.v` | **16bit・16入力セレクタ**: 16個のレジスタから1つを選択する。1bit MUXを16個並列化。 |
| **Level 1** | `register_file_4to16_dec.v` | **書き込みデコーダ**: 4bitの `rd` 番号を16本の書き込み許可信号に変換する。 |
| **Level 2** | `register_file_16bit_reg.v` | **単位レジスタ**: 書き込み有効信号(EN)付きの16bitレジスタ。 |
| **Level 2** | `register_file_1bit_mux_16to1.v` | **単位セレクタ**: 16個の1bit入力から1つを選択する最小単位。 |

---

### 2. 各モジュールの詳細説明

#### 【Level 2】最小構成要素（Primitives）
- **`register_file_16bit_reg.v`**
    - 役割: 16bitのDフリップフロップ群。書き込み有効信号がHighのときのみ値を更新します。
- **`register_file_1bit_mux_16to1.v`**
    - 役割: 4bitの選択信号に基づき、16個の1bit入力から1本を抽出します。
    - 視覚化効果: 回路図上で、読み出しポートが「16対1」の集約構造であることを示します。

#### 【Level 1】機能ブロック（Functional Blocks）
- **`register_file_bank.v`**
    - 役割: **generate文を用いて `register_file_16bit_reg` を15個配置**します（R1〜R15用）。
    - R0の扱い: R0に対してはレジスタを配置せず、常に `16'h0000` を出力するように配線します。
    - 視覚化効果: 回路図上で R0 だけが配線のみ（定数0）であり、R1以降がレジスタの箱として並んでいる様子がISAの「R0=0」の制約と一致して見えます。

- **`register_file_read_mux.v`**
    - 役割: **generate文を用いて `register_file_1bit_mux_16to1` を16個配置**します。
    - 視覚化効果: 16bitのデータバスがビットごとに独立したセレクタに入力される様子が示され、ビット並列処理の構造が理解しやすくなります。

- **`register_file_4to16_dec.v`**
    - 役割: `rd_addr` と `RegWrite` 信号を組み合わせ、どのレジスタのEN端子を叩くかを決定します。
    - 視覚化効果: 「どのレジスタに書き込むか」という制御の流れが、デコーダから各レジスタへ伸びる16本の信号線として視覚化されます。

#### 【Level 0】最上位（Top Module）
- **`register_file.v`**
    - 役割: 2組の `read_mux`（RS1用とRS2用）と、1つの `bank`、1つの `dec` を接続します。
    - 視覚化効果: 「アドレス入力」→「デコーダ/セレクタ」→「中央のレジスタ群」という、レジスタファイルの全体像が完成します。

---

### 3. この階層化によるメリット

1.  **R0の特殊性が一目でわかる**:
    ソースコード上の `if (addr == 0)` という記述ではなく、回路図上で「R0の場所には箱（レジスタ）がなく、地面（GND/0）に繋がっている」ことが見えるため、教育的効果が高いです。
2.  **ポートの独立性の理解**:
    RS1用とRS2用のMUXが物理的に2つ存在することを示すことで、1クロックで2つのレジスタを同時に読み出せる（ハーバードアーキテクチャのレジスタ版）仕組みが理解できます。
3.  **合成効率と構造化の両立**:
    巨大な `case` 文で書くと合成ツールが複雑なロジックに最適化してしまいがちですが、MUXをビットごとに分割して構造化することで、規則正しい回路配置（視覚的な美しさ）を維持できます。
