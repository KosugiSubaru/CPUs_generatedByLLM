「alu_core（ALU中核）」モジュールを、論理合成後の回路図で「演算の種類（算術・論理・シフト）とその選択」がISAの定義通りに視覚化されるよう階層化・パターン構造化します。

この設計では、**「算術演算（加減算）」「論理演算」「シフト演算」**を独立したブロックとして設計し、最後に16ビット幅の巨大なセレクタで結果を抽出する構造をとります。

---

### 1. 階層構造の概要

| 階層レベル | ファイル名 | 役割 |
| :--- | :--- | :--- |
| **Level 0** | `alu_core.v` | **最上位統合**: 算術、論理、シフトの各ユニットを統合し、フラグ（Z, N, V）を生成。 |
| **Level 1** | `alu_core_nbit_adder_subtractor.v` | **算術ユニット**: 16bit加減算器。V(オーバーフロー)とN(負)フラグの元を生成。 |
| **Level 1** | `alu_core_nbit_logic.v` | **論理ユニット**: bitwiseのAND, OR, XOR, NOTを並列実行。 |
| **Level 1** | `alu_core_nbit_shifter.v` | **シフトユニット**: 算術右シフト(SRA)と算術左シフト(SLA)を実行。 |
| **Level 1** | `alu_core_nbit_result_mux.v` | **結果選択セレクタ**: 演算結果の中から、命令(opcode)に応じて1つを選択。 |
| **Level 2** | `alu_core_1bit_full_adder.v` | **算術最小単位**: 1bitの全加算器。 |
| **Level 2** | `alu_core_1bit_logic.v` | **論理最小単位**: 1bitに対する論理演算（AND/OR/XOR/NOT）。 |
| **Level 2** | `alu_core_1bit_result_mux.v` | **選択最小単位**: 1bitに対する結果選択用多入力MUX。 |

---

### 2. 各モジュールの詳細説明

#### 【Level 2】最小構成要素（Primitives）
- **`alu_core_1bit_full_adder.v`**
    - 役割: キャリー伝播を伴う1bit加算を行います。
- **`alu_core_1bit_logic.v`**
    - 役割: AとBの入力に対し、AND, OR, XOR, NOTのすべての結果を同時に算出します。
- **`alu_core_1bit_result_mux.v`**
    - 役割: オペコードに応じた選択信号を受け取り、算術・論理・シフトの候補から1bit分を抽出します。

#### 【Level 1】Nビット・パターン構造（N-bit Functional Units）
- **`alu_core_nbit_adder_subtractor.v`**
    - 役割: **generate文を用いて `alu_core_1bit_full_adder` を16個連結**します。
    - 減算処理: 減算命令(opcode:0001)のとき、B入力を反転してキャリーインに1を入力（2の補数）する論理を視覚化します。
    - 視覚化効果: 加算と減算が同じ「加算器チェーン」を通る様子が見て取れます。
- **`alu_core_nbit_logic.v`**
    - 役割: **generate文を用いて `alu_core_1bit_logic` を16個並列配置**します。
    - 視覚化効果: ビットごとに独立して論理演算が行われる「論理ゲートの壁」として表示されます。
- **`alu_core_nbit_shifter.v`**
    - 役割: バレルシフタ構造をとります。ビット位置の入れ替え（配線）の組み合わせでシフトを実現します。
    - 視覚化効果: 配線が複雑に交差する様子から、シフト演算の物理的なイメージが伝わります。
- **`alu_core_nbit_result_mux.v`**
    - 役割: **generate文を用いて `alu_core_1bit_result_mux` を16個配置**します。
    - 視覚化効果: 膨大な演算候補から最終的な `rd` への書き込み値が1つに絞り込まれる「収束点」として視覚化されます。

#### 【Level 0】ALU最上位（Top Module）
- **`alu_core.v`**
    - 役割: 上記ユニットをすべて並列に動かし、その結果からフラグを計算します。
    - フラグ生成の視覚化:
        - `N`: 結果の15bit目から直接配線。
        - `Z`: 全ビットのNORゲート（または比較器）。
        - `V`: ISA定義通りのMSBキャリー論理。

---

### 3. この階層化によるメリット

1.  **演算の並列性の理解**:
    CPU内部では「加算器が動いている横で、論理演算器も同時に計算している」が、最後にMUXで選ばれなかった方は捨てられる、というハードウェア特有の動作が回路図から直感的に理解できます。
2.  **フラグと演算の分離**:
    演算結果そのものと、その結果の状態を示す「フラグ」がどのように生成されるのか、その配線の流れを追うことができます。
3.  **パタン構造による美しさ**:
    1bit単位のモジュールを generate で並べることで、合成後の回路が非常に規則正しく、教科書的なレイアウト（データパスの視覚化）になります。
