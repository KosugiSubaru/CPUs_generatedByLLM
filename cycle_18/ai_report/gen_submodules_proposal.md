提供されたISA定義とアーキテクチャに基づき、教育用シングルサイクルCPUを構成する1階層下のサブモジュール群と、その動作原理を提案します。

---

### 1. 提案するサブモジュール構成

CPU内部を以下の7つの主要モジュールに分割することを提案します。

#### ① PC (Program Counter)
- **役割**: 次に実行する命令のメモリアドレス（`o_addr_to_imem`）を保持します。
- **動作**: クロックの立ち上がりごとに、後述の「Next PC Logic」から送られてくる次のアドレスに更新されます。

#### ② Control Unit (デコーダ)
- **役割**: 命令（`i_instr`）の下位4ビット（opcode）を解析し、CPU全体の動きを制御する信号を生成します。
- **主な出力**: 
    - ALUの演算種別、レジスタ書き込み有効フラグ、データメモリ書き込み有効フラグ（`o_dmem_wen`）、ソースセレクタ信号（ALUに渡すのはレジスタか即値か）など。

#### ③ Register File (16x16bit)
- **役割**: 16個の16ビットレジスタ（R0〜R15）を管理します。
- **動作**: 
    - `rs1`, `rs2` アドレスに基づき2つのデータを非同期で読み出します。
    - `rd` アドレスに対し、書き込み有効信号がある時にクロック同期でデータを書き込みます。
    - **R0 (zero_reg)** は常に `0` を出力し、書き込みを無視する論理を含みます。

#### ④ Immediate Generator (即値生成器)
- **役割**: 命令語内の各フィールドから即値（`imm`）を抽出し、16ビットに**符号拡張**します。
- **動作**: 命令の種類（I-type, Load/Store, Branch, Jump）に応じて、切り出すビット位置を変更します。

#### ③ ALU (Arithmetic Logic Unit)
- **役割**: 算術演算、論理演算、シフト演算を実行し、計算結果と演算フラグ（Z, N, V）を出力します。
- **フラグ**: 
    - **Z (Zero)**: 結果が0
    - **N (Negative)**: 結果が負（MSBが1）
    - **V (Overflow)**: 符号付き溢れ

#### ⑥ Flag Register (ステータスレジスタ)
- **役割**: **「条件分岐は1クロック前のフラグを参照する」**という仕様を満たすため、ALUが出力したフラグを1クロック保持します。
- **動作**: 毎クロック、ALUからのZ, N, Vフラグを保存し、次の命令の分岐判定に提供します。

#### ⑦ Next PC Logic (分岐制御ユニット)
- **役割**: 次のPCの値を決定します。
- **動作**: 
    - 通常時：`PC + 2`
    - 条件分岐（BLT/BZ）一致時：`PC + imm`
    - JAL：`PC + imm`
    - JALR：`rs1 + imm`

### 2. CPUの動作プロセス（シングルサイクルの流れ）

1クロックの間で、以下の処理が連鎖的（組合せ回路的）に行われ、クロックの立ち上がりで状態が確定します。

1.  **Instruction Fetch (命令フェッチ)**
    - `PC`が出力するアドレスにより `imem` から命令（`i_instr`）が読み出されます。
2.  **Decode & Register Read (デコードと読み出し)**
    - `Control Unit` が命令を解析し、各種制御信号を出力します。
    - 同時に `Register File` から `rs1`, `rs2` の値が読み出されます。
    - `Immediate Generator` が命令から即値を生成します。
3.  **Execute (実行)**
    - `ALU` が、レジスタ値または即値を用いて演算を行います。
    - 条件分岐命令の場合、`Flag Register` に保存されている**前回のフラグ**を見て、分岐するかどうかを判定します。
4.  **Memory Access (データメモリアクセス)**
    - `load` 命令なら `dmem` からデータを読み出し、`store` 命令なら `dmem` への書き込み信号（`o_dmem_wen`）を有効にします。
5.  **Write Back (書き戻し)**
    - `ALUの計算結果`、`メモリからの読み出しデータ`、または `PC+2 (JAL系)` のいずれかを選択し、`Register File` の `rd` に書き込みます。
    - 同時に、新しい演算フラグが `Flag Register` に保存され、`PC` が更新されます。

### 教育的ポイント
この構成にすることで、「命令のビットがどのように分解され（デコード）」、「どのようにデータが流れ（データパス）」、「結果がどこに保存されるか」を、各モジュールの入出力を観測することで直感的に理解できるようになります。特に、**Flag Register**を独立させることで、ISAの「1クロック前のフラグを参照する」という特徴的な仕様を視覚化できます。
