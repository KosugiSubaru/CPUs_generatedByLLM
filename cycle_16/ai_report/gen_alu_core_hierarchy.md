教育用CPUの核心部である`alu_core`において、演算の種類（算術・論理・シフト）を機能ブロックとして分離し、さらにデータパスの「ビットスライス（1ビットごとの繰り返し）」構造を視覚化する構成を提案します。

合成後の回路図で、「計算の内容」と「16ビット幅の広がり」が直感的に理解できる設計にします。

### 1. 提案する階層構造とファイル一覧

| 階層レベル | ファイル名 | 役割 |
| :--- | :--- | :--- |
| **L0 (Top)** | `alu_core.v` | 最上位モジュール。各機能ブロックを統合し、最終結果とフラグを出力する。 |
| **L1 (Functional)** | `alu_core_arithmetic_16bit.v` | 16ビットの加減算を行う。フラグ（Z, N, V）の計算も担当する。 |
| | `alu_core_logic_16bit.v` | ビットごとの論理演算（AND, OR, XOR, NOT）を一括で行う。 |
| | `alu_core_shifter_16bit.v` | 算術シフト（SRA, SLA）を行う。 |
| | `alu_core_mux_8to1_16bit.v` | 演算結果の中から、命令（Opcode）に応じて1つを選択する。 |
| **L2 (Pattern)** | `alu_core_full_adder.v` | 1ビットの全加算器。算術ユニットの最小単位。 |
| | `alu_core_logic_slice.v` | 1ビット分の論理演算ゲート群。論理ユニットの最小単位。 |
| | `alu_core_mux_8to1_1bit.v` | 1ビット幅の8入力セレクタ。出力MUXの最小単位。 |

---

### 2. 各モジュールの役割と階層化の意図

#### ① 算術演算の視覚化 (`arithmetic`, `full_adder`)
加算と減算を一つの「16ビット加算器」に統合します。減算時は入力を反転してキャリーインを1にすることで、**「2の補数」**による計算を回路図上で明示します。`full_adder`を16個直列に並べることで、キャリー（繰り上がり）が下位ビットから上位ビットへ伝播する様子を視覚化します。

#### ② 論理演算のビットスライス (`logic`, `logic_slice`)
`AND`, `OR`, `XOR`, `NOT` はビット間で独立しているため、`logic_slice`という1ビット分のゲートの塊を定義し、それを16個並列に並べます。これにより、データパス全体で同じ論理操作が並列に行われている概念を視覚的に強調します。

#### ③ シフト演算の分離 (`shifter`)
算術右シフト(SRA)と算術左シフト(SLA)を独立したブロックにします。Verilogの演算子（`>>>`, `<<<`）は論理合成時に強力な最適化がかかりますが、モジュールとして独立させることで、回路図上では「ビットをずらす配線の束」として認識しやすくなります。

#### ④ 出力選択の構造化 (`mux_8to1_16bit`, `mux_8to1_1bit`)
ALUの最終出力を決めるマルチプレクサを、1ビット幅のセレクタが16個並んだ構造として記述します。これにより、「16ビットの演算結果が、セレクタによって一斉に切り替えられる」というデータフローを視覚化します。

---

### 3. 教育的な利点

1.  **「算術」と「論理」の分離**: 数学的な計算（繰り上がりがある）と、デジタル論理的な計算（ビットごとに独立）の違いを、ブロックの構造として理解できます。
2.  **フラグ生成の根拠**: なぜMSB（最上位ビット）が負フラグ(N)になるのか、なぜ加算結果が0のときにゼロフラグ(Z)が立つのかを、配線の接続先から学べます。
3.  **計算の並列性**: `generate` 文による16個の同一モジュールのインスタンス化は、現代のプロセッサが16ビットや32ビットのデータを「一度に」処理できる物理的な理由（ハードウェアが並列に存在するから）を視覚的に示します。
