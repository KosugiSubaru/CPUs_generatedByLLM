教育用CPUにおいて、プログラムの実行順序を制御する`next_pc_logic`は、プログラムカウンタ（PC）の「次の値」を決定する組み合わせ回路の塊です。

ISAで定義された「PC+2」「条件分岐」「ジャンプ」の各挙動を、独立した加算器とマルチプレクサ（MUX）のブロックとして分離し、それらが選択信号によって切り替わる様子を視覚化する階層設計を提案します。

### 1. 提案する階層構造とファイル一覧

| 階層レベル | ファイル名 | 役割 |
| :--- | :--- | :--- |
| **L0 (Top)** | `next_pc_logic.v` | 最上位モジュール。複数の加算器と選択ロジックを統合し、最終的な次アドレスを出力する。 |
| **L1 (Functional)** | `next_pc_logic_adder_16bit.v` | 16ビットの加算器。次アドレスの候補（PC+2, PC+imm, rs1+imm）の算出に使用。 |
| | `next_pc_logic_mux_4to1_16bit.v` | 4つの候補から次PCを1つ選択する16ビット幅MUX。内部で1ビット幅MUXを16個並列化。 |
| **L2 (Pattern)** | `next_pc_logic_full_adder.v` | 1ビットの全加算器。加算器ユニットの最小単位。 |
| | `next_pc_logic_mux_4to1_1bit.v` | 1ビット幅の4入力セレクタ。MUXユニットの最小単位。 |

---

### 2. 各モジュールの役割と階層化の意図

#### ① ISAの挙動と加算器の対応 (`next_pc_logic.v`, `adder_16bit`)
ISAの定義に基づき、3つの独立した加算計算を並列に配置します。
- **Adder 1**: `PC + 2`（通常の命令実行）
- **Adder 2**: `PC + imm`（`blt`, `bz`, `jal` 命令用）
- **Adder 3**: `rs1 + imm`（`jalr` 命令用）
これらを個別のブロックとして回路図に見せることで、「命令の種類によって異なる加算パスが使われる」というCPUの動作原理を視覚化します。

#### ② ビットスライスによるデータパスの表現 (`mux_4to1_16bit`, `mux_4to1_1bit`)
- 16ビットの選択回路を1つ作るのではなく、1ビット幅のセレクタ（L2）を16個並列に並べる（L1）構造にします。
- 合成後の回路図では、16ビットのバスが16個の同じ部品に一斉に入力され、一斉に切り替わる様子が確認できます。これにより、ハードウェアにおける「並列処理」と「データパス幅」の概念を直感的に理解させます。

#### ③ 条件分岐と選択ロジックの統合
`next_pc_logic.v` (L0) は、デコーダーからの「ジャンプ命令か？」という信号と、フラグ判定部からの「分岐条件は成立したか？」という信号を組み合わせ、MUXの選択信号（`pc_sel`）を生成します。
- `00`: PC + 2
- `01`: PC + imm (分岐成立 または JAL)
- `10`: rs1 + imm (JALR)
これにより、論理回路がどのように「判断」を下し、次の一手（アドレス）を選んでいるかを学習者に示します。

---

### 3. 教育的な利点

1.  **アドレス計算の物理的実体**: プログラム上の「ジャンプ」や「分岐」が、ハードウェア的には「加算器によるオフセット計算」と「MUXによる経路選択」の組み合わせに過ぎないことを理解できます。
2.  **規則性とパターンの認識**: `generate` 文による繰り返し構造を通じて、大規模なデジタル回路が小さな部品の規則的な並びで構成されていることを学べます。
3.  **データフローの追跡**: 合成後の回路図で、現在のPC値から次PC値へ至る複数のパスを追いかけることが容易になり、シングルサイクルCPUの「1クロックですべてが完結する」という特徴を把握しやすくなります。
