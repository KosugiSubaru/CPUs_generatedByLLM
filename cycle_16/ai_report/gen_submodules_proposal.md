教育用シングルサイクルCPUとして、命令セット（ISA）とハードウェア構成を直感的に結びつけるための、1階層下のサブモジュール構成を提案します。

---

### 1. 提案するサブモジュール一覧

CPUモジュールの内部を以下の7つの主要モジュールに分割することを提案します。

| モジュール名 | 役割 |
| :--- | :--- |
| **PC (Program Counter)** | 次に実行する命令のアドレス（`w_addr_to_imem`）を保持・更新する。 |
| **Decoder (Control Unit)** | 命令（`i_instr`）を解読し、各部への制御信号（ALUの演算種別、レジスタ書き込み許可など）を生成する。 |
| **Register File** | 16個の16bitレジスタを管理する。2つの読み出しポートと1つの書き込みポートを持つ。R0は常に0。 |
| **ImmGen (Immediate Generator)** | 命令の種類に応じて、異なるビット位置にある即値を抽出し、16ビットに符号拡張する。 |
| **ALU (Arithmetic Logic Unit)** | 算術論理演算（加減算、論理演算、シフト）を実行し、計算結果とフラグ（Z, N, V）を出力する。 |
| **Flag Register** | ALUが出力したフラグを1クロック保持する。「1クロック前のフラグを参照する」という仕様を実現するために必要。 |
| **Next PC Logic** | 通常のインクリメント（PC+2）に加え、分岐（Branch）やジャンプ（Jump）の条件に応じて次のPC値を決定する。 |

---

### 2. 各モジュールの相互作用と動作フロー

シングルサイクルCPUとして、1クロック内で以下のステップが同時に進行します。

#### ① Fetch (命令取得)
- **PC**が現在の命令アドレスを命令メモリ(imem)へ送ります。
- **imem**から現在の命令（`i_instr`）が返ってきます。

#### ② Decode & Read (解読と読み出し)
- **Decoder**が命令の`opcode`を見て、以下の制御信号を出力します。
    - ALUで何をするか（ADDかSUBか等）
    - レジスタに書き込むかどうか
    - メモリ(dmem)の読み書きをどうするか
- **Register File**から`rs1`、`rs2`番指定のアドレスのデータが読み出されます。
- **ImmGen**が命令内のビットを組み替え、即値（`imm`）を生成します。

#### ③ Execute (演算)
- **ALU**が「レジスタの値」や「即値」を使い演算を行います。
- **ALU**は計算結果とともに、ゼロ(Z)・負(N)・オーバーフロー(V)のフラグを生成します。
- **Flag Register**は、このフラグを保存します（この値は**次の命令**の分岐判定に使われます）。

#### ④ Memory & Write Back (メモリ実行と書き込み)
- `load`/`store`命令の場合、ALUの計算結果をアドレスとして**dmem**へアクセスします。
- **Register File**の`rd`に対して、以下のいずれかのデータが書き込まれます。
    - ALUの計算結果（算術演算命令など）
    - メモリからの読み出しデータ（`load`命令）
    - PC+2の値（`jal`/`jalr`命令：戻り先アドレスの保存）
    - 即値そのもの（`loadi`命令）

#### ⑤ Next PC (次アドレスの決定)
- **Next PC Logic**が次のPCを決定します。
    - 通常：`PC + 2`
    - 分岐命令（`blt`, `bz`）：**Flag Register**に保存されている「1つ前の命令のフラグ」を確認し、条件を満たせば `PC + imm`。
    - ジャンプ命令（`jal`, `jalr`）：指定されたジャンプ先アドレス。

---

### 3. この構成の教育的ポイント

1.  **フラグの遅延参照**: `Flag Register`を独立させることで、ISA仕様にある「1クロック前のフラグを参照する」という動作を視覚的に（フリップフロップとして）理解しやすくなります。
2.  **即値の抽出**: 命令形式によって`imm`のビット位置がバラバラ（11:4だったり15:4だったり）であるため、`ImmGen`というモジュールを設けることで、ハードウェアでの「ビットの配線（配線替え）」の概念を学べます。
3.  **データフローの可視化**: ALUの出力が「メモリのアドレス」になる場合（load/store）と、「レジスタへの書き込みデータ」になる場合があることを示すことで、バスとセレクタ（マルチプレクサ）の役割が明確になります。
