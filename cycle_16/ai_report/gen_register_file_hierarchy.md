教育用CPUとして、16個のレジスタが整然と並び、それらを「デコーダー」と「マルチプレクサ（MUX）」で制御する構造を視覚化します。特に「R0は常に0」というISAの制約を、回路構造として明示的に分離して表現することを提案します。

### 1. 提案する階層構造とファイル一覧

| 階層レベル | ファイル名 | 役割 |
| :--- | :--- | :--- |
| **L0 (Top)** | `register_file.v` | 最上位モジュール。デコーダー、セル配列、読み出しMUXを統合する。 |
| **L1 (Functional)** | `register_file_cell_array.v` | 16個の16bitレジスタセルを並べた配列。R0（定数0）とR1-R15（可変）の差を回路で示す。 |
| | `register_file_mux_16to1_16bit.v` | 16個の入力から1つを選ぶ16bit幅のMUX。内部で1bit幅MUXを16個並列化する。 |
| | `register_file_decoder_4to16.v` | 4bitの書き込みアドレスを16本の有効信号（One-hot）に変換する。 |
| **L2 (Pattern)** | `register_file_cell_16bit.v` | 16ビットのフリップフロップ群。書き込み許可信号（EN）付き。 |
| | `register_file_mux_16to1_1bit.v` | 1ビット幅の16入力セレクタ。パタン構造化の最小単位。 |

---

### 2. 各モジュールの役割と階層化の意図

#### ① 書き込み制御の視覚化 (`decoder_4to16`)
4ビットのアドレス入力から16本の独立した信号線を生成します。これにより、論理合成後の回路図で「どのアドレスが指定されたときに、どのレジスタの書き込み許可（EN）がアクティブになるか」が、1対1の物理的な配線として確認できます。

#### ② レジスタセルの構造化 (`cell_array`, `cell_16bit`)
- `register_file_cell_array.v` 内で `generate` 文を使用し、15個の通常レジスタ（R1-R15）をインスタンス化します。
- **R0の特殊性**: R0については `cell_16bit` を使わず、単に `16'h0000` を定数として出力する配線にすることで、「R0は書き込み不可で常に0である」というISAの仕様を、物理的な回路構成（フリップフロップが存在しないこと）として学習者に示します。

#### ③ 読み出しパスのパタン化 (`mux_16to1_16bit`, `mux_16to1_1bit`)
- 16対1の大きなセレクタを1つ作るのではなく、まず「1ビット幅の16入力セレクタ」を定義します。
- `register_file_mux_16to1_16bit.v` でこの1ビット版を `generate` 文を用いて16個横に並べます。
- 合成後の回路図では、16個の同じ構造のMUXブロックが綺麗に並ぶため、ビットスライス構造（データの各ビットが同じように処理される様子）が視覚的に強調されます。

#### ④ ポート構成の明確化
- シングルサイクルCPUに対応するため、2つの読み出しポート（rs1, rs2）と1つの書き込みポート（rd）を独立して持ち、これらが内部でどう配線されているかを L0 で示します。

---

### 3. 教育的な利点

1.  **「アドレス」から「配線」への変換**: デコーダーを通すことで、バイナリ値（0010）が物理的な場所（3番目のレジスタ）に変換される仕組みを理解できます。
2.  **空間的な理解**: 16個のブロックが並んでいる様子を見ることで、レジスタファイルが「メモリ（記憶場所）の集合体」であることを実感できます。
3.  **ビット並列性の理解**: `generate` 文による横並びのインスタンス化により、16bitのデータが16本の平行なパスを通るコンピュータの基本構造を学べます。
