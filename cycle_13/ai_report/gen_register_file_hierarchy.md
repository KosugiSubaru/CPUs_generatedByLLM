「register_file（レジスタファイル）」の階層化とパターン構造化を提案します。

レジスタファイルは「16個の16ビットレジスタ」と、それらを選択する「デコーダー」および「大きなマルチプレクサ（MUX）」で構成されます。これらをビット単位から階層化することで、データの保存と選択の仕組みを視覚的に理解しやすくします。

---

### 1. 階層構造の定義

「1ビットの記憶素子」を束ねて「16ビットの単語」を作り、さらにそれを「16個並べる」という、コンピュータ構成の王道的なパターンを回路に反映させます。

| 階層レベル | ファイル名 | 役割 |
| :--- | :--- | :--- |
| **Level 3 (Leaf)** | `register_file_bit.v` | 1ビットのD-FF。書き込み許可信号（EN）付きの記憶最小単位。 |
| | `register_file_mux2_1bit.v` | 1ビットの2入力MUX。大規模MUXを構成する基本部品。 |
| **Level 2 (Structure)** | `register_file_word.v` | `register_file_bit` を **generate文で16個並列化**。16ビットのレジスタ1つ分。 |
| | `register_file_mux16_1bit.v` | `register_file_mux2_1bit` を組み合わせて作る、1ビット幅の16入力MUX。 |
| | `register_file_decoder.v` | 4ビットのアドレスを16本の書き込み許可信号（ワンホット）に変換するデコーダー。 |
| **Level 1 (Top-slice)** | `register_file_mux16_16bit.v` | `register_file_mux16_1bit` を **generate文で16個並列化**。16ビット幅の16入力MUX。 |
| **Level 0 (Top)** | `register_file.v` | 16個の `register_file_word` とデコーダー、読み出し用MUXを組み合わせた全体モジュール。 |

---

### 2. 各ファイルの役割と階層レベルの詳細

#### (1) register_file_bit.v (Level 3)
もっとも小さな記憶単位です。書き込み有効（EN）が1のときだけ、クロックの立ち上がりでデータを取り込みます。

#### (2) register_file_word.v (Level 2)
`register_file_bit` を16個並べたものです。CPUが扱う「1語（Word）」の単位を形成します。
- **教育的意図**: 回路図で、16ビットのデータバスが16個のフリップフロップに分配される様子を視覚化します。

#### (3) register_file_mux16_1bit.v (Level 2)
16個のレジスタから、指定されたアドレスの1ビットを選び出します。
- **構造**: `mux2` を階層的に組み合わせたツリー構造（8→4→2→1）として実装し、大規模選択回路の構成パターンを示します。

#### (4) register_file_mux16_16bit.v (Level 1)
`register_file_mux16_1bit` を16個（ビット分）並列に配置します。
- **手法**: `generate` 文を使用。
- **教育的意図**: 「16入力MUX」という巨大な機能が、実は「各ビットごとに独立した選択器があるだけ」であることを回路図で示します。

#### (5) register_file_decoder.v (Level 2)
書き込み先アドレス（rd）を解析し、16個ある `register_file_word` のうち、どのレジスタの書き込み許可信号（EN）を有効にするか決定します。
- **特記**: ISA定義に基づき、レジスタ0（zero_reg）への書き込みを常に無効にする論理をここに含めます。

#### (6) register_file.v (Level 0)
全ての要素を統合します。
- **記憶部**: `generate` 文を用いて `register_file_word` を16個インスタンス化します。
- **読み出し部**: 2つのポート（rs1, rs2）のために、`register_file_mux16_16bit` を2つ配置します。
- **レジスタ0の扱い**: レジスタ0の出力は常に0になるよう、MUXの入力0番に定数0を供給します。

---

### 3. この構成による教育的メリット

1.  **「ビット」から「語」へのスケールアップ**: 
    小さな1ビットの素子が集まって16ビットのレジスタになり、それが16個集まってレジスタファイルになる過程がファイル構成と回路図で一目で分かります。
2.  **配線の視覚化**: 
    16個のレジスタそれぞれから、読み出し用MUXへと「16ビット×16本」の配線が伸びる様子は、CPU内部のデータ移動を理解する上で非常に強力な視覚教材となります。
3.  **レジスタ0の特殊性の理解**: 
    回路図上で、レジスタ0だけが書き込み信号を受け取らない、あるいは出力が定数に固定されている様子を見ることで、ISAの制約が物理的にどう実現されているかを理解できます。
