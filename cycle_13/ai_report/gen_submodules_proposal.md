教育用シングルサイクルCPUを構成するために、1階層下のサブモジュール（構成要素）を以下に提案します。

この構成は、命令の実行フロー（Fetch, Decode, Execute, Memory, Write Back）を視覚的に追いやすくすることを重視しています。

---

### 1. CPU構成サブモジュール一覧

| モジュール名 | 役割・機能説明 |
| :--- | :--- |
| **PC (Program Counter)** | 次に実行する命令のメモリアドレス（`w_addr_to_imem`）を保持するレジスタです。 |
| **Control Unit** | `opcode`を解析し、ALUの演算種別、レジスタ書き込み許可、メモリ書き込み許可、セレクタの切り替え信号などを生成する「CPUの司令塔」です。 |
| **Register File** | 16ビット×16個のレジスタ群です。2つの読み出しポート（`rs1`, `rs2`）と1つの書き込みポート（`rd`）を持ち、番号0は常に0を出力します。 |
| **ALU (Arithmetic Logic Unit)** | 加減算、論理演算、シフト演算を行う演算器です。演算結果とともに、フラグ（Z, N, V）を出力します。 |
| **Immediate Generator** | 命令内の即値フィールド（4bit, 8bit, 12bitなど）を抽出し、16ビットに符号拡張するユニットです。 |
| **Flag Register** | ALUが出力したフラグ（Z, N, V）を1クロック保持します。このCPUの仕様である「1クロック前のフラグを参照する」ために必要です。 |
| **Branch Control Unit** | フラグレジスタの値と命令を確認し、分岐条件（`blt`, `bz`）が成立しているか判定して、次のPC値を決定します。 |
| **Mux (Multiplexer) 群** | ALUの入力（レジスタ値か即値か）や、レジスタへの書き込みデータ（ALU結果、メモリ値、またはPC+2か）を選択するスイッチです。 |

---

### 2. CPUの動作プロセス（相互作用の説明）

シングルサイクルCPUのため、以下の1～5の工程が1クロック（`i_clk`の立ち上がり）の間に一気に駆け抜けます。

#### 1. 命令フェッチ (Instruction Fetch)
- **PC**が現在の命令アドレスを命令メモリ（`imem`）へ送ります。
- `imem`から16ビットの命令コード（`i_instr`）が読み出されます。
- 同時に、次の命令アドレスの候補として「PC + 2」が計算されます。

#### 2. デコードとレジスタ読み出し (Decode & Register Read)
- **Control Unit**が命令の下位4ビット（`opcode`）を見て、どの制御信号を有効にするか決定します。
- **Register File**から、命令で指定された`rs1`, `rs2`番地のデータが読み出されます。
- **Immediate Generator**が、命令内の即値を符号拡張して準備します。

#### 3. 実行とフラグ更新 (Execute)
- **ALU**が、レジスタ値や即値を用いて演算（加算、論理演算、シフト等）を行います。
- 演算結果は、データメモリのアドレスやレジスタへの書き込みデータとして使われます。
- **重要:** ALUが生成した新しいフラグ（Z, N, V）は、クロックの立ち上がりで**Flag Register**に保存され、**次の命令**で参照可能になります。

#### 4. メモリアクセス (Memory Access)
- `load`命令の場合：ALUで計算したアドレス（`rs1 + imm`）を使って**dmem**からデータを読み出します。
- `store`命令の場合：`w_dmem_wen`を有効にし、`rs2`の内容をメモリに書き込みます。

#### 5. ライトバックとPC更新 (Write Back & PC Update)
- **Register File**の`rd`に対し、演算結果、メモリからの読み出しデータ、またはリンクアドレス（PC+2）のいずれかを書き込みます。
- **Branch Control Unit**は、保持されている「前回のフラグ」を見て条件分岐が成立するか判定します。成立していれば分岐先アドレス、そうでなければ「PC + 2」をPCにセットし、次のサイクルへ備えます。

---

### 教育的なポイント
この階層構造にすることで、学習者は「命令メモリから入ってきたビット列が、どのようにして各モジュールのスイッチ（セレクタ）を切り替え、データがレジスタや演算器を流れていくか」を、回路図上で左から右への流れとして視覚的に理解できるようになります。
