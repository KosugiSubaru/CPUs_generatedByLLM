「alu（演算器）」の階層化とパターン構造化を提案します。

ALUは、算術演算（加減算）、論理演算、シフト演算の3つの主要な演算器と、最終的な結果を選択するマルチプレクサ、そしてフラグ（Z, N, V）を計算する論理回路で構成されます。これらを機能ごとにブロック化し、さらにビット単位の構造（Ripple Carry等）を視覚化します。

---

### 1. 階層構造の定義

算術演算の「キャリーの伝播」や、論理演算の「ビット並列性」を回路図で確認できるように、以下の4レベルで構成します。

| 階層レベル | ファイル名 | 役割 |
| :--- | :--- | :--- |
| **Level 3 (Leaf)** | `alu_full_adder.v` | 1ビット全加算器。加減算の最小構成単位。 |
| | `alu_logic_bit.v` | 1ビット分の論理演算（AND, OR, XOR, NOT）を行う最小単位。 |
| **Level 2 (Structure)** | `alu_adder_16bit.v` | `alu_full_adder` を **generate文で16個接続**。加算および減算（2の補数制御付き）を実行。 |
| | `alu_logic_16bit.v` | `alu_logic_bit` を **generate文で16個並列化**。16ビットの論理演算を実行。 |
| **Level 1 (Unit)** | `alu_shifter_16bit.v` | 16ビットの算術シフト（SRA, SLA）を実行する演算ユニット。 |
| | `alu_mux8_16bit.v` | 8つの演算結果候補から、オペコードの下位3bitに従って1つを選択。 |
| **Level 0 (Top)** | `alu.v` | 上記ユニットを統合し、演算結果の出力とフラグ（Z, N, V）の最終計算を行う。 |

---

### 2. 各ファイルの役割と階層レベルの詳細

#### (1) alu_full_adder.v (Level 3)
算術演算の心臓部です。A, B, Carry-Inを入力し、SumとCarry-Outを出力します。

#### (2) alu_adder_16bit.v (Level 2)
`alu_full_adder` を16個数珠つなぎにした Ripple Carry Adder です。
- **教育的意図**: 減算時に「Bを反転してCarry-Inを1にする」という2の補数の仕組みを、インスタンスへの入力操作として回路図上で明示します。
- **視覚的効果**: 16個の加算器ブロックが横に並び、キャリー信号が右から左へ流れる様子が確認できます。

#### (3) alu_logic_16bit.v (Level 2)
AND, OR, XOR, NOTの各ゲートをビットごとにまとめたユニットです。
- **教育的意図**: 論理演算がビット間で独立して並列実行される性質を、16個の独立した `alu_logic_bit` ブロックとして視覚化します。

#### (4) alu_shifter_16bit.v (Level 1)
`rs1` を `rs2` 分だけシフトします。シフト演算は複雑な配線になりやすいため、教育用として「左シフト」と「右シフト（符号拡張あり）」の切り替えを明確にした独立ブロックとします。

#### (5) alu_mux8_16bit.v (Level 1)
各種演算ユニットの出力から、現在のオペコードに対応するものを選びます。
- **選択肢**: Add, Sub, AND, OR, XOR, NOT, SRA, SLA の8種類。

#### (6) alu.v (Level 0)
全体の司令塔です。
- **フラグ計算**: 演算結果から `Z` (Zero) と `N` (Negative) を計算します。
- **オーバーフロー計算**: ISA定義にある通り、加減算時のみ有効な `V` (Overflow) フラグの複雑な条件（符号ビットの不一致）を、専用の組み合わせ論理として実装します。

---

### 3. この構成による教育的メリット

1.  **演算の分類学**:
    「算術」「論理」「シフト」というCPUの主要な演算カテゴリが、回路図上でそれぞれ独立した物理的なブロック（島）として見えるようになります。
2.  **フラグの由来の理解**:
    Zフラグが「全ビットのORの反転」であることや、Vフラグが「演算前の符号と後の符号の比較」から来ていることが、最終出力段のゲートとして視覚的に把握できます。
3.  **データパスの分岐と収束**:
    入力データが一度に全ての演算器（Adder, Logic, Shifter）に配られ、最後にMUXで1つに絞られるという、シングルサイクルCPU特有の「投機的な演算実行」の様子を学ぶことができます。
