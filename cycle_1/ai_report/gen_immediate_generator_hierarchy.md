「immediate_generator」モジュールを、命令形式ごとの即値構成（ビット位置の組み換えと符号拡張）が視覚的に理解できるよう、以下の階層構造で提案します。

この構成では、各命令形式（I, S, L, B型）に応じた符号拡張回路を個別のブロックとして視覚化し、それらを最終的に1つのMUXで選択する様子を回路図上で表現します。

---

### 1. 階層構造とファイル一覧

| 階層レベル | ファイル名 | 役割 |
| :--- | :--- | :--- |
| **Level 0 (基本素子)** | `immediate_generator_sign_ext_bit.v` | 1ビット分の処理単位。入力ビットをそのまま通すか、符号ビットで埋めるかを決定する素子。 |
| | `immediate_generator_mux_4to1_1bit.v` | 4つの形式の中から1つを選択する1ビット幅のセレクタ。 |
| **Level 1 (構造化ユニット)** | `immediate_generator_i_extender.v` | **I型(4bit)** 用。`generate`文を用い、命令の[7:4]を符号拡張して16bitにする。 |
| | `immediate_generator_s_extender.v` | **S型(4bit)** 用。`generate`文を用い、命令の[15:12]を符号拡張して16bitにする。 |
| | `immediate_generator_l_extender.v` | **L型(8bit)** 用。`generate`文を用い、命令の[11:4]を符号拡張して16bitにする。 |
| | `immediate_generator_b_extender.v` | **B型(12bit)** 用。`generate`文を用い、命令の[15:4]を符号拡張して16bitにする。 |
| | `immediate_generator_mux_4to1_16bit.v` | `generate`文を用い、16ビット幅で各形式の即値を選択するモジュール。 |
| **Level 2 (サブモジュール・トップ)** | `immediate_generator.v` | 形式ごとのエクステンダーを並列に配置し、制御信号に基づき最終的な即値を出力するトップ。 |

### 2. 各モジュールの詳細説明

#### Level 0: 基本素子
*   **immediate\_generator\_sign\_ext\_bit.v**
    *   **役割**: 即値の各ビットを生成する最小単位。そのビット位置が「元の即値が存在する範囲」なら入力をそのまま出力し、「符号拡張が必要な範囲」なら符号ビット（MSB）を出力します。
*   **immediate\_generator\_mux\_4to1\_1bit.v**
    *   **役割**: セレクタの最小単位。

#### Level 1: パターン構造化（形式別の拡張ロジック）
*   **immediate\_generator\_[i/s/l/b]\_extender.v**
    *   **役割**: 各命令形式（I, S, L, B）に特化した符号拡張器です。
    *   **回路図での見え方**: 命令デコーダから届く「生のビット列」が、このブロックを通ることで「16ビットの符号付き整数」に変換される様子が4つの独立した経路として視覚化されます。
*   **immediate\_generator\_mux\_4to1\_16bit.v**
    *   **役割**: 16ビットの4対1マルチプレクサです。
    *   **回路図での見え方**: 4つの拡張器からの出力が1点に集まり、制御信号によって1つが選ばれる様子が見えます。

#### Level 2: サブモジュール・トップ
*   **immediate\_generator.v**
    *   **役割**: 命令デコーダから抽出された生のビット（`i_imm_raw`）と、現在の命令がどの形式であるかの選択信号を受け取り、16bit即値を出力します。

---

### 3. 教育的なポイント：視覚化のメリット

1.  **「符号拡張」の物理的な理解**: 
    「上位ビットをMSBで埋める」という操作が、`sign_ext_bit` が並列に並び、特定のビット（符号ビット）が複数のビット位置に分配されている配線として視覚化されます。
2.  **ISA定義と回路の直結**: 
    「addi命令はI型だから、この経路（`i_extender`）を通る」ということが、回路図上の信号の流れとして追跡可能になります。
3.  **ビット位置の組み換えの明示**: 
    `store`命令（S型）のように即値が命令コードの離れた位置にある場合でも、専用のエクステンダーを用意することで、どのビットがどこへ移動して16bit整数を構成するかが明確になります。

このように各形式を独立したブロックとして定義することで、ISAのドキュメントに記載されている「即値のビット構成図」と実際の回路図が1対1で対応し、学習者が理解しやすい構造となります。
