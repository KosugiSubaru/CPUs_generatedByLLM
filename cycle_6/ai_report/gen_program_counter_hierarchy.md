「視覚的に回路構成を理解する」という教育的目的に基づき、`program_counter`（プログラムカウンタ）とその周辺の次アドレス生成論理を、構成要素（レジスタ、加算器、セレクタ）ごとに階層化・パターン構造化します。

この設計では、単純なPCレジスタだけでなく、ISAで定義されている「PC+2」「PC+imm (Branch/JAL)」「rs1+imm (JALR)」の計算に必要な論理をPCユニットに集約し、それらをビットレベルの再利用可能なパーツから構成します。

---

### 1. 階層構造とファイル構成

以下の6つのファイル（モジュール）に分割します。

| レベル | ファイル名 | 役割の説明 |
| :--- | :--- | :--- |
| **L3 (Top)** | `program_counter_top.v` | PCユニットの最上位。adderやmuxを組み合わせて「次アドレス」を決定・保持する。 |
| **L2 (Functional)** | `program_counter_register_16bit.v` | 16ビットのD-FF群。generate文を用いて1bit DFFから構成する。 |
| **L2 (Functional)** | `program_counter_adder_16bit.v` | 16ビットの加算器。generate文を用いてフルアダーから構成する。 |
| **L2 (Functional)** | `program_counter_mux_4to1_16bit.v` | 次のPCを選択する4入力セレクタ（通常、分岐、JALR、停止など）。 |
| **L1 (Primitive)** | `program_counter_dff.v` | 非同期リセット付きの1ビットD-フリップフロップ。 |
| **L1 (Primitive)** | `program_counter_full_adder.v` | 1ビットのフルアダー（全加算器）。 |

---

### 2. サブモジュールの提案内容

#### ① program_counter_full_adder.v (L1)
加算器の最小単位です。
*   **役割**: 1ビットのA, Bおよびキャリー入力を受け取り、和とキャリー出力を返します。
*   **視覚化のポイント**: 加算論理の最小単位として回路図に現れます。

#### ② program_counter_dff.v (L1)
レジスタの最小単位です。
*   **役割**: クロック立ち上がりに同期して入力DをQに出力します。
*   **視覚化のポイント**: 1ビットの値を保持する素子として個別に認識できます。

#### ③ program_counter_adder_16bit.v (L2)
*   **役割**: `program_counter_full_adder` を **generate文で16個インスタンス化** し、リップルキャリー方式で結合します。
*   **ISAとの対応**: `PC + 2` や `PC + imm` の計算に使用します。
*   **パターン構造化**: 1ビットの積み重ねで16ビットが作られる様子を視覚化します。

#### ④ program_counter_register_16bit.v (L2)
*   **役割**: `program_counter_dff` を **generate文で16個インスタンス化** します。
*   **ISAとの対応**: 現在実行中の命令アドレス（PC）を保持します。

#### ⑤ program_counter_mux_4to1_16bit.v (L2)
*   **役割**: 制御信号に基づき、以下の4つから次のPCを選択します。
    1.  `PC + 2`（通常実行）
    2.  `PC + imm`（分岐成立、JAL）
    3.  `rs1 + imm`（JALR）
    4.  `PC`（リセット直後やホールド用）
*   **視覚化のポイント**: プログラムの実行フローがどのように分岐するかを選択する「交差点」として機能します。

#### ⑥ program_counter_top.v (L3)
*   **役割**: 上記モジュールをすべて結合し、`cpu`モジュールにPC値（`o_addr_to_imem`）を提供します。
*   **内部構成**:
    *   インスタンス1: `adder_pc_plus_2` (PCに定数2を加算)
    *   インスタンス2: `adder_target` (PCまたはrs1に即値を加算)
    *   インスタンス3: `next_pc_mux` (条件に応じて次アドレスを選択)
    *   インスタンス4: `pc_reg` (確定した次アドレスをクロック同期で保存)

---

### 3. 本構成による動作の相互作用

1.  **通常時**: `adder_pc_plus_2` が常に `PC+2` を計算し、`mux` を通じて `pc_reg` に送られます。
2.  **分岐命令時**: `Control Unit` からの信号と `Flag Register` の状態に基づき、分岐成立なら `adder_target` が計算した `PC+imm` が `mux` で選択されます。
3.  **JALR命令時**: レジスタから読み出された `rs1` と `imm` を `adder_target` で加算し、その結果が `mux` を経由して `pc_reg` に格納されます。
4.  **リセット**: `i_rst_n` がLowになると `pc_reg` 内の全ての `dff` が 0 に初期化され、0番地からの実行が開始されます。

このように、各計算パス（加算器）と選択肢（MUX）を分離して階層化することで、回路図を見ただけで「どこでアドレスが計算され、どう選ばれているか」が直感的に理解できるようになります。
