教育用シングルサイクルCPUを構成するために、1階層下のサブモジュール（機能ブロック）を以下のように提案します。この構成は、データパスの流れを視覚的に追いやすく、ISAの各命令（演算、メモリアクセス、分岐）との対応が明確になるよう設計しています。

---

### 1. 提案するサブモジュール構成

CPU内部を以下の7つの主要モジュールに分割します。

#### ① PC Unit (プログラムカウンタ・ユニット)
*   **役割**: 次に実行する命令のアドレスを保持・更新します。
*   **詳細**: 通常は `PC + 2` を行いますが、分岐（Branch）成立時やジャンプ（JAL/JALR）時には、計算された飛び先アドレスを選択します。

#### ② Control Unit (制御ユニット / デコーダ)
*   **役割**: 命令（`instr[3:0]` のopcode等）を解析し、CPU全体のセレクタや書き込み許可信号（`RegWrite`, `MemWrite`, `ALUOp` 等）を生成します。
*   **詳細**: 「この命令はレジスタに書き込むか？」「ALUでどの演算を行うか？」といった司令塔の役割を果たします。

#### ③ Register File (レジスタファイル)
*   **役割**: 16個の16ビットレジスタ（R0～R15）を保持します。
*   **詳細**: 2つのレジスタを同時に読み出し（rs1, rs2）、1つのレジスタ（rd）に書き込みます。**R0は常に0を返す**特殊な論理を内包します。

#### ④ Immediate Generator (即値生成器)
*   **役割**: 命令形式（I-type, Jump系など）に応じて、命令語の中にあるビット列を抽出し、16ビットに**符号拡張**します。
*   **詳細**: 命令によって即値のビット位置が異なる（4bit, 8bit, 12bit）ため、これらを適切に整列させます。

#### ⑤ ALU (算術論理演算ユニット)
*   **役割**: 加減算、論理演算、シフト演算を実際に実行します。
*   **詳細**: 演算結果とともに、フラグ（Z:ゼロ、N:負、V:オーバーフロー）を出力します。

#### ⑥ Flag Register (フラグ・レジスタ)
*   **役割**: ALUが出力したフラグ（Z, N, V）を**1クロック保持**します。
*   **詳細**: 本ISAの「条件分岐は1クロック前のフラグを参照する」という仕様を実現するために必須となる、同期更新されるレジスタです。

#### ⑦ Data Path Muxes (データパス・セレクタ群)
*   **役割**: ALUへの入力（レジスタ値か即値か）や、レジスタへの書き込みデータ（ALU結果かメモリ値かPC+2か）を切り替えます。

---

### 2. CPUの動作プロセスとモジュール間の相互作用

シングルサイクルCPUのため、1クロックの間に以下の「フェッチ〜書き戻し」がすべて完了します。

1.  **Instruction Fetch (命令取り出し)**:
    `PC Unit`が出力するアドレス（`w_addr_to_imem`）に基づき、命令メモリから命令（`i_instr`）を読み出します。
2.  **Decode & Register Read (解読と読み出し)**:
    *   `Control Unit`が命令をデコードし、各種制御信号を出力します。
    *   `Register File`は、命令中の `rs1`, `rs2` フィールドに対応するデータを読み出します。
    *   `Immediate Generator`は、命令中の即値を16ビットに拡張します。
3.  **Execute (演算)**:
    *   `ALU`は、レジスタ値や即値を用いて演算（add, sub, or 等）を行います。
    *   演算結果に基づき、新しいフラグ（Z, N, V）が生成されます。
    *   **分岐判定**: `Flag Register`に保存されている「前回のフラグ」と命令の種類に基づき、条件分岐（`blt`, `bz`）を成立させるか判定します。
4.  **Memory Access (データメモリアクセス)**:
    *   `load` 命令の場合、ALU結果をアドレスとしてデータメモリから値を読み出します。
    *   `store` 命令の場合、`i_wen`を有効にしてデータをメモリに書き込みます。
5.  **Write Back (書き戻し)**:
    *   最終的な結果（ALUの結果、メモリからの読み出しデータ、あるいはJAL用のPC+2）を、`rd` で指定されたレジスタに書き込みます。
6.  **State Update (状態更新)**:
    *   クロックの立ち上がりで、`PC` が次の値（PC+2 or 分岐先）に更新されます。
    *   同時に、今回のALUフラグが `Flag Register` に保存されます。

### 視覚的なデータの流れ（簡易図）
```text
[PC Unit] ──> (Instruction Memory) ──> [Control Unit]
                                   │          │
                                   ├──> [Register File] ──┐
                                   │          │           V
[Immediate Gen] ───────────────────┼──────────┴────> [  ALU  ] ──> (Data Memory)
                                   │                      │             │
                                   └──────────────────────┴──> [Mux] <──┘
                                                                │
                                     [Register File] <──────────┘ (Write Back)
```

この階層構造により、各モジュールがISAのどの仕様（符号拡張、ゼロレジスタ、フラグレジスタ等）を担っているかが明確になり、教育用として理解しやすい構成となります。
