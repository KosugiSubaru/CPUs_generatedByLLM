提供されたISA定義とシステムアーキテクチャに基づき、教育用シングルサイクルCPUを構成する1階層下のサブモジュール群と、その動作原理を提案します。

---

### 1. CPUを構成するサブモジュール一覧

シングルサイクルCPUとして、1クロック内で「命令フェッチ、デコード、実行、メモリ・レジスタ書き込み」を完結させるための構成です。

| モジュール名 | 役割・機能説明 |
| :--- | :--- |
| **PC (Program Counter)** | 次に実行する命令のメモリアドレス（`w_addr_to_imem`）を保持するレジスタ。 |
| **Control Unit (Decoder)** | 命令の下位4ビット（`opcode`）を解析し、ALUの演算種別、レジスタ書き込み有効化、メモリ書き込み有効化（`o_dmem_wen`）などの制御信号を生成する。 |
| **Register File** | 16ビット×16本のレジスタ群。2つの読み出しポート（`rs1`, `rs2`）と1つの書き込みポート（`rd`）を持つ。R0は常に0を出力する。 |
| **ALU (Arithmetic Logic Unit)** | 加減算、論理演算（AND/OR/XOR/NOT）、算術シフトを実行する。演算結果とともに、フラグ（Z, N, V）を算出する。 |
| **Immediate Generator** | 命令内の各ビットフィールドから即値（`imm`）を抽出し、符号拡張を行って16ビットデータに変換する。 |
| **Flag Register** | ALUが算出したフラグ（Z, N, V）を1クロック保持する。ISAの仕様通り「1クロック前の結果」を参照するために必須。 |
| **Branch/Jump Logic** | 条件分岐（`blt`, `bz`）やジャンプ（`jal`, `jalr`）の成否を判定し、次のPC値を「PC+2」にするか「分岐先アドレス」にするか選択する。 |
| **Data Mux (セレクタ群)** | ALUへの入力（`rs2` か `imm` か）、レジスタファイルへの書き込みデータ（`ALU出力` か `メモリ出力` か `PC+2` か）などを選択する。 |

---

### 2. CPUの動作プロセスとモジュール間の相互作用

シングルサイクルCPUは、1つのクロックサイクルの間に以下のパスをデータが流れます。

#### ① 命令フェッチ (Fetch)
`PC`が出力するアドレスを命令メモリ（`imem`）に送り、命令コード（`i_instr`）を取得します。同時に、標準的な次アドレスとして「`PC + 2`」を計算しておきます。

#### ② デコードとレジスタ読み出し (Decode)
`Control Unit`が命令を解析し、`Register File`から`rs1`と`rs2`のデータを読み出します。同時に`Immediate Generator`が命令中のビットから符号拡張された即値を生成します。

#### ③ 実行とフラグ更新 (Execute)
`ALU`は、制御信号に従って演算（`rs1 + rs2` や `rs1 + imm` など）を行います。
*   **フラグ処理:** この時算出されたZ, N, Vフラグは、`Flag Register`に送られ、**次のクロックの立ち上がり**で保存されます。
*   **分岐判定:** 分岐命令（`blt`, `bz`）は、**現在の`Flag Register`（＝1クロック前の演算結果）**を見て、分岐するかどうかを判定します。

#### ④ メモリ・レジスタ書き込み (Memory/Write-back)
*   **Load/Store:** `load`命令なら`dmem`からデータを読み出し、`store`命令なら`dmem`へデータを書き込みます（`o_dmem_wen`を1にする）。
*   **Register Write:** 最終的な演算結果、またはメモリから読み出したデータを`Register File`の`rd`に書き込みます。

#### ⑤ 次のPCの決定 (Next PC)
`Branch/Jump Logic`の結果に基づき、次のクロックで`PC`にセットする値を決定します。
*   通常時：`PC + 2`
*   分岐・ジャンプ時：`pc + imm` または `rs1 + imm`

---

### 教育的視点でのポイント
この構成では、**「フラグが1クロック遅れて参照される」**というISAの特殊な仕様が、`Flag Register`という物理的なレジスタの存在として可視化されます。これにより、回路上のタイミング（データの依存関係）を学ぶ教材として非常に適した構造となります。
